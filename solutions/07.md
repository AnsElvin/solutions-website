# 题目07：三数之和

## 题目描述

看着小鲁AC了第一题，小华接着提出第二问：

给定一个目标值 target，请在整数数组a中，找出三个元素(x,y,z)使x+y+z==target。

请找到所有满足条件的三元组，并且请按从小到大的顺序输出所有合法的三元组。

注意：三元组中不允许包含重复数字，且输出的三元组中要求 x<y<z。

例如：给定target=17，n=7，数组a=[0,2,7,10,10,15,18,25]
结果返回两个三元组：(0,2,15), (2,7,10)

### 输入

输入数据为2行，第一行有两个整数target和n，其中target代表要搜索的目标和，n表示数组a的元素个数

第二行表示数组a的n个数，每个元素用空格隔开。

### 输出

输出所有满足和为target的三元组(x,y,z)，要求(x<y<z)并且不允许有重复数字。

把三元组按照x的大小升序输出，x相同的按照y的大小升序输出。

### 输入样例

```
17 7
0 2 7 10 10 15 18 25
```

### 输出样例

```
0 2 15
0 7 10
```

## 题目分析

这是两数之和的进阶版，需要在数组中找到所有满足条件的三元组，并且要去重。

**关键信息提取：**
- 数组可能包含重复元素
- 需要找到所有满足条件的三元组
- 三元组中元素必须满足 x<y<z（不允许重复）
- 按x升序，x相同按y升序输出
- 输入样例中有重复元素，需要去重

**考察要点：**
1. **双指针算法扩展**：如何将两数之和扩展到三数之和
2. **去重策略**：如何处理重复元素
3. **排序应用**：如何利用排序简化问题
4. **时间复杂度优化**：O(n²)解法

## 详细思路推演

### 第一步：理解问题

我们需要找到所有满足 x+y+z = target 的三元组，其中x<y<z。数组可能包含重复元素，但输出的三元组不能重复。

### 第二步：思考如何转化为两数之和

对于每个元素a[i]作为x，我们需要在后面的元素中找到两个数y和z，使得 y+z = target - x。

这就转化为了两数之和问题，但需要注意：
- 要遍历所有可能的x
- 需要去重
- 需要找到所有解

### 第三步：排序的重要性

先对数组排序，可以带来以下好处：
1. 方便去重（相同元素相邻）
2. 可以使用双指针法
3. 输出天然有序

### 第四步：去重策略

1. **跳过重复的x**：如果a[i] == a[i-1]，跳过（避免重复的三元组）
2. **跳过重复的y**：找到解后，如果a[left] == a[left+1]，跳过
3. **跳过重复的z**：找到解后，如果a[right] == a[right-1]，跳过

### 第五步：算法流程

```
1. 排序数组
2. 遍历每个元素a[i]作为x：
   a. 如果i>0且a[i]==a[i-1]，跳过（去重）
   b. 使用双指针在[i+1, n-1]范围内找y和z：
      i. left = i+1, right = n-1
      ii. 如果a[left]+a[right] == target-a[i]：
          - 记录三元组
          - 跳过重复的left
          - 跳过重复的right
          - left++, right--
      iii. 如果a[left]+a[right] < target-a[i]：left++
      iv. 如果a[left]+a[right] > target-a[i]：right--
3. 输出所有三元组
```

## 分步算法讲解

### 算法步骤

1. **排序数组**：将数组按升序排列
2. **遍历第一个元素**：for (i = 0; i < n-2; i++)
3. **去重x**：if (i > 0 && a[i] == a[i-1]) continue
4. **双指针初始化**：left = i+1, right = n-1
5. **双指针查找**：
   - 如果sum == target-a[i]：记录解，去重，移动指针
   - 如果sum < target-a[i]：left++
   - 如果sum > target-a[i]：right--

### 流程图

```
开始
  ↓
读取输入，排序数组
  ↓
遍历 i = 0 到 n-3
  ↓
a[i] == a[i-1]? → 是 → continue
  ↓ 否
left = i+1, right = n-1
  ↓
循环：left < right
  ↓
sum = a[left] + a[right]
  ↓
sum == target-a[i]? → 是 → 记录解
                           ↓
                     跳过重复left/right
                           ↓
                     left++, right--
  ↓ 否
sum < target-a[i]? → 是 → left++
  ↓ 否
sum > target-a[i]? → 是 → right--
  ↓
继续循环
  ↓
结束遍历
  ↓
输出所有三元组
  ↓
结束
```

## C语言代码

```c
#include <stdio.h>
#include <stdlib.h>

int cmp(const void *a, const void *b) {
    return *(int*)a - *(int*)b;
}

int main() {
    int target, n;
    scanf("%d %d", &target, &n);

    int *a = (int*)malloc(n * sizeof(int));
    for (int i = 0; i < n; i++) {
        scanf("%d", &a[i]);
    }

    qsort(a, n, sizeof(int), cmp);

    for (int i = 0; i < n - 2; i++) {
        if (i > 0 && a[i] == a[i-1]) continue;

        int left = i + 1;
        int right = n - 1;
        int need = target - a[i];

        while (left < right) {
            int sum = a[left] + a[right];
            if (sum == need) {
                printf("%d %d %d\n", a[i], a[left], a[right]);

                while (left < right && a[left] == a[left+1]) left++;
                while (left < right && a[right] == a[right-1]) right--;

                left++;
                right--;
            } else if (sum < need) {
                left++;
            } else {
                right--;
            }
        }
    }

    free(a);
    return 0;
}
```

## 常见错误与易错点

1. **忘记排序**：双指针法依赖数组有序，必须先排序。

2. **去重不彻底**：
   - 忘记跳过重复的x：会导致重复的三元组
   - 忘记跳过重复的y/z：会导致重复的三元组

3. **边界条件错误**：
   - 循环条件应为 `i < n-2`（至少保留两个元素给y和z）
   - 双指针循环条件应为 `left < right`

4. **输出格式错误**：题目要求输出元素值，不是下标。

5. **时间复杂度**：暴力枚举O(n³)会超时，必须使用O(n²)的双指针法。

## 复杂度分析

### 时间复杂度

**O(n²)**

- 排序：O(n log n)
- 外层循环：O(n)
- 内层双指针：O(n)
- 总体：O(n²)

### 空间复杂度

**O(log n) 或 O(n)**

- 排序的递归栈空间：O(log n)
- 如果使用额外数组存储结果：O(n)

## 扩展思考

### 扩展1：四数之和

类似思路，固定两个数，用双指针找另外两个：

```cpp
vector<vector<int>> fourSum(vector<int>& nums, int target) {
    sort(nums.begin(), nums.end());
    vector<vector<int>> result;
    
    for (int i = 0; i < nums.size(); i++) {
        if (i > 0 && nums[i] == nums[i-1]) continue;
        
        for (int j = i + 1; j < nums.size(); j++) {
            if (j > i + 1 && nums[j] == nums[j-1]) continue;
            
            int left = j + 1, right = nums.size() - 1;
            int need = target - nums[i] - nums[j];
            
            while (left < right) {
                int sum = nums[left] + nums[right];
                if (sum == need) {
                    result.push_back({nums[i], nums[j], nums[left], nums[right]});
                    while (left < right && nums[left] == nums[left+1]) left++;
                    while (left < right && nums[right] == nums[right-1]) right--;
                    left++;
                    right--;
                } else if (sum < need) {
                    left++;
                } else {
                    right--;
                }
            }
        }
    }
    return result;
}
```

**时间复杂度**：O(n³)，空间复杂度：O(log n)

### 扩展2：k数之和（递归解法）

可以递归地将问题分解为(k-1)数之和：

```cpp
vector<vector<int>> kSum(vector<int>& nums, int target, int k, int start) {
    vector<vector<int>> result;
    
    if (k == 2) {
        int left = start, right = nums.size() - 1;
        while (left < right) {
            int sum = nums[left] + nums[right];
            if (sum == target) {
                result.push_back({nums[left], nums[right]});
                while (left < right && nums[left] == nums[left+1]) left++;
                while (left < right && nums[right] == nums[right-1]) right--;
                left++;
                right--;
            } else if (sum < target) {
                left++;
            } else {
                right--;
            }
        }
        return result;
    }
    
    for (int i = start; i < nums.size(); i++) {
        if (i > start && nums[i] == nums[i-1]) continue;
        
        vector<vector<int>> temp = kSum(nums, target - nums[i], k - 1, i + 1);
        for (auto& t : temp) {
            t.insert(t.begin(), nums[i]);
            result.push_back(t);
        }
    }
    
    return result;
}

// 使用示例
vector<vector<int>> threeSum(vector<int>& nums) {
    sort(nums.begin(), nums.end());
    return kSum(nums, 0, 3, 0);
}
```

**时间复杂度**：O(n^(k-1))，空间复杂度：O(k)（递归栈）

### 扩展3：哈希表解法

对于三数之和，可以用哈希表存储两个数的和，但去重比较麻烦：

```cpp
vector<vector<int>> threeSumHash(vector<int>& nums) {
    sort(nums.begin(), nums.end());
    vector<vector<int>> result;
    unordered_map<int, vector<pair<int, int>>> hash;
    
    for (int i = 0; i < nums.size(); i++) {
        for (int j = i + 1; j < nums.size(); j++) {
            hash[nums[i] + nums[j]].push_back({i, j});
        }
    }
    
    for (int k = 0; k < nums.size(); k++) {
        int target = -nums[k];
        if (hash.count(target)) {
            for (auto& p : hash[target]) {
                int i = p.first, j = p.second;
                if (k < i) {
                    vector<int> triplet = {nums[k], nums[i], nums[j]};
                    // 需要去重
                    bool duplicate = false;
                    for (auto& r : result) {
                        if (r[0] == triplet[0] && r[1] == triplet[1] && r[2] == triplet[2]) {
                            duplicate = true;
                            break;
                        }
                    }
                    if (!duplicate) {
                        result.push_back(triplet);
                    }
                }
            }
        }
    }
    
    return result;
}
```

**时间复杂度**：O(n²)，空间复杂度：O(n²)

### 扩展4：不允许排序的情况

如果不允许修改原数组，可以使用哈希表，但时间复杂度更高：

```cpp
vector<vector<int>> threeSumNoSort(vector<int>& nums) {
    vector<vector<int>> result;
    unordered_set<string> seen;
    
    for (int i = 0; i < nums.size(); i++) {
        unordered_set<int> hash;
        for (int j = i + 1; j < nums.size(); j++) {
            int complement = -nums[i] - nums[j];
            if (hash.count(complement)) {
                vector<int> triplet = {nums[i], nums[j], complement};
                sort(triplet.begin(), triplet.end());
                string key = to_string(triplet[0]) + "," + to_string(triplet[1]) + "," + to_string(triplet[2]);
                if (!seen.count(key)) {
                    seen.insert(key);
                    result.push_back(triplet);
                }
            }
            hash.insert(nums[j]);
        }
    }
    
    return result;
}
```

**时间复杂度**：O(n²)，空间复杂度：O(n²)

### 扩展5：三数之和最接近

如果需要找到和最接近target的三个数：

```cpp
int threeSumClosest(vector<int>& nums, int target) {
    sort(nums.begin(), nums.end());
    int closest = nums[0] + nums[1] + nums[2];
    
    for (int i = 0; i < nums.size(); i++) {
        if (i > 0 && nums[i] == nums[i-1]) continue;
        
        int left = i + 1, right = nums.size() - 1;
        while (left < right) {
            int sum = nums[i] + nums[left] + nums[right];
            if (abs(sum - target) < abs(closest - target)) {
                closest = sum;
            }
            if (sum < target) {
                left++;
            } else {
                right--;
            }
        }
    }
    
    return closest;
}
```

### 扩展7：三数之和等于0（优化版）

针对三数之和等于0的特殊情况，可以进行剪枝优化：

```cpp
vector<vector<int>> threeSumZero(vector<int>& nums) {
    sort(nums.begin(), nums.end());
    vector<vector<int>> result;
    
    for (int i = 0; i < nums.size(); i++) {
        if (nums[i] > 0) break;  // 剪枝：第一个数大于0，后面都是正数，不可能和为0
        if (i > 0 && nums[i] == nums[i-1]) continue;
        
        int left = i + 1, right = nums.size() - 1;
        while (left < right) {
            int sum = nums[i] + nums[left] + nums[right];
            if (sum == 0) {
                result.push_back({nums[i], nums[left], nums[right]});
                while (left < right && nums[left] == nums[left+1]) left++;
                while (left < right && nums[right] == nums[right-1]) right--;
                left++;
                right--;
            } else if (sum < 0) {
                left++;
            } else {
                right--;
            }
        }
    }
    
    return result;
```

## 总结

这道题是两数之和的进阶版，核心思想是将三数之和转化为两数之和问题。

**解题要点：**
1. 先排序数组（O(n log n)）
2. 遍历每个元素作为第一个数x
3. 使用双指针法在剩余部分找y和z
4. 注意去重：跳过重复的x、y、z
5. 时间复杂度O(n²)，空间复杂度O(log n)
