# 题目02：完美立方

## 题目描述

形如 a³ = b³ + c³ + d³ 的等式被称为完美立方等式。例如 12³ = 6³ + 8³ + 10³。编写一个程序，对任给的正整数 N (N ≤ 100)，寻找所有的四元组 (a, b, c, d)，使得 a³ = b³ + c³ + d³，其中 a, b, c, d 大于 1，小于等于 N，且 b ≤ c ≤ d。

### 输入

一个正整数 N (N ≤ 100)。

### 输出

每行输出一个完美立方。输出格式为：

```
Cube = a, Triple = (b,c,d)
```

其中 a, b, c, d 所在位置分别用实际求出四元组值代入。

请按照 a 的值，从小到大依次输出。当两个完美立方等式中 a 的值相同，则 b 值小的优先输出、仍相同则 c 值小的优先输出、再相同则 d 值小的先输出。

### 输入样例

```
24
```

### 输出样例

```
Cube = 6, Triple = (3,4,5)
Cube = 12, Triple = (6,8,10)
Cube = 12, Triple = (6,8,10)
Cube = 18, Triple = (2,12,16)
Cube = 18, Triple = (9,12,15)
Cube = 18, Triple = (9,12,15)
Cube = 19, Triple = (3,10,18)
Cube = 20, Triple = (7,14,17)
Cube = 24, Triple = (12,16,20)
Cube = 24, Triple = (12,16,20)
```

## 题目分析

这是一道典型的枚举类题目，需要遍历所有可能的四元组 (a, b, c, d)，检查是否满足完美立方等式。

**关键信息提取：**
- 等式：a³ = b³ + c³ + d³
- 约束条件：2 ≤ a, b, c, d ≤ N，且 b ≤ c ≤ d
- 输出顺序：按 a 从小到大，a 相同按 b 从小到大，以此类推
- 数据范围：N ≤ 100，100³ = 1,000,000

**考察要点：**
1. **枚举算法**：如何高效地遍历所有可能的组合
2. **优化策略**：如何减少不必要的计算
3. **输出格式**：如何按照指定格式输出结果

## 详细思路推演

### 第一步：理解问题

题目要求我们找到所有满足 a³ = b³ + c³ + d³ 的四元组，其中 a, b, c, d 都是大于等于2、小于等于N的整数，并且 b ≤ c ≤ d。

### 第二步：思考枚举范围

最直接的思路是使用四层循环来枚举所有可能的四元组：

1. **a 的范围**：从 2 到 N
2. **b 的范围**：从 2 到 N
3. **c 的范围**：从 b 到 N（因为 b ≤ c）
4. **d 的范围**：从 c 到 N（因为 c ≤ d）

### 第三步：思考如何检查条件

对于每一个四元组 (a, b, c, d)，我们需要检查是否满足 a³ = b³ + c³ + d³。

直接计算立方值并比较：
- 计算 a³ = a * a * a
- 计算 b³ + c³ + d³
- 如果两者相等，则找到了一个完美立方

### 第四步：思考优化策略

四层循环的时间复杂度是 O(N⁴)，对于 N = 100，最坏情况下需要执行 100⁴ = 100,000,000 次循环，这虽然在计算机能力范围内，但可以通过优化来减少不必要的计算：

1. **提前计算立方值**：在循环内部预先计算 a³、b³、c³、d³，避免重复计算
2. **提前终止**：当 b³ > a³ 时，b³ + c³ + d³ 肯定大于 a³，可以直接跳出 b 的循环
3. **提前终止**：当 b³ + c³ > a³ 时，可以直接跳出 c 的循环
4. **提前终止**：当 b³ + c³ + d³ > a³ 时，可以直接跳出 d 的循环

### 第五步：思考输出格式

按照题目要求，输出格式为：
```
Cube = a, Triple = (b,c,d)
```

需要注意输出顺序：
- 按 a 从小到大
- a 相同按 b 从小到大
- b 相同按 c 从小到大
- c 相同按 d 从小到大

由于我们的枚举顺序本身就是按这个顺序进行的，所以直接输出即可，不需要额外排序。

## 分步算法讲解

### 算法步骤

1. **读取输入**：读取正整数 N
2. **枚举 a**：从 2 到 N 遍历 a
3. **计算 a³**：预先计算 a 的立方值
4. **枚举 b**：从 2 到 N 遍历 b
5. **计算 b³**：预先计算 b 的立方值
6. **检查 b³**：如果 b³ > a³，跳出 b 的循环
7. **枚举 c**：从 b 到 N 遍历 c
8. **计算 c³**：预先计算 c 的立方值
9. **检查 b³ + c³**：如果 b³ + c³ > a³，跳出 c 的循环
10. **枚举 d**：从 c 到 N 遍历 d
11. **计算 d³**：预先计算 d 的立方值
12. **检查条件**：如果 b³ + c³ + d³ == a³，输出结果
13. **检查和**：如果 b³ + c³ + d³ > a³，跳出 d 的循环

### 流程图

```
开始
  ↓
读取 N
  ↓
a = 2 到 N
  ↓
计算 a³
  ↓
b = 2 到 N
  ↓
计算 b³
  ↓
b³ > a³? → 是 → 跳出 b 循环
  ↓ 否

  ↓
计算 c³
  ↓
b³ + c³ > a³? → 是 → 跳出 c 循环
  ↓ 否

  ↓
计算 d³
  ↓
b³ + c³ + d³ == a³? → 是 → 输出结果
  ↓

  ↓
结束
```

## C语言代码

```c
#include <stdio.h>

int main() {
    int N;                      // 存储输入的正整数 N
    scanf("%d", &N);            // 读取输入
    
    // 第一层循环：枚举 a，从 2 到 N
    for (int a = 2; a <= N; a++) {
        int a3 = a * a * a;     // 预先计算 a 的立方值，避免重复计算
        
        // 第二层循环：枚举 b，从 2 到 N
        for (int b = 2; b <= N; b++) {
            int b3 = b * b * b; // 预先计算 b 的立方值
            
            // 优化：如果 b³ > a³，那么 b³ + c³ + d³ 肯定大于 a³
            // 可以直接跳出 b 的循环，因为 b 是递增的
            if (b3 > a3) {
                break;
            }
            
            // 第三层循环：枚举 c，从 b 到 N（因为 b ≤ c）
            for (int c = b; c <= N; c++) {
                int c3 = c * c * c; // 预先计算 c 的立方值
                
                // 优化：如果 b³ + c³ > a³，那么 b³ + c³ + d³ 肯定大于 a³
                // 可以直接跳出 c 的循环，因为 c 是递增的
                if (b3 + c3 > a3) {
                    break;
                }
                
                // 第四层循环：枚举 d，从 c 到 N（因为 c ≤ d）
                for (int d = c; d <= N; d++) {
                    int d3 = d * d * d; // 预先计算 d 的立方值
                    
                    // 检查是否满足完美立方等式
                    if (b3 + c3 + d3 == a3) {
                        // 按照指定格式输出结果
                        printf("Cube = %d, Triple = (%d,%d,%d)\n", a, b, c, d);
                    } else if (b3 + c3 + d3 > a3) {
                        // 优化：如果和已经大于 a³，可以直接跳出 d 的循环
                        break;
                    }
                }
            }
        }
    }
    
    return 0;                   // 程序正常结束
}
```

## 常见错误与易错点

1. **枚举范围错误**：b 应该从 2 开始（因为 b > 1），如果从 1 开始会多枚举无用的情况。

2. **缺少约束条件**：应该保证 b ≤ c ≤ d，否则会重复输出相同的四元组（如 (3,4,5) 和 (4,3,5) 是同一个解）。

3. **没有提前终止优化**：当 b³ + c³ + d³ > a³ 时可以 break，继续循环只会让结果更大，浪费时间。

4. **输出格式错误**：注意题目要求的输出格式，包括空格、等号、括号等，必须严格匹配。

5. **数据类型溢出**：如果 N 很大，立方运算可能超出 int 范围，需要改用 long long 类型。

## 复杂度分析

### 时间复杂度

**O(N⁴)**，四层循环嵌套。

- 第一层循环：N 次（a 从 2 到 N）
- 第二层循环：平均 N/2 次（b 从 2 到 N，但有提前终止）
- 第三层循环：平均 N/3 次（c 从 b 到 N，但有提前终止）
- 第四层循环：平均 N/4 次（d 从 c 到 N，但有提前终止）

实际运行中，由于有多个提前终止条件，实际执行次数远小于 N⁴。对于 N = 100，优化后的代码运行速度非常快。

### 空间复杂度

**O(1)**，常数空间复杂度。

- 只使用了几个变量：N, a, b, c, d, a3, b3, c3, d3
- 变量数量固定，与输入规模无关

## 扩展思考

### 扩展1：优化算法（减少循环层数）

可以先计算所有数的立方值，存储在一个数组中，避免重复计算：

```c
#include <stdio.h>

int main() {
    int N;
    scanf("%d", &N);
    
    // 预先计算所有数的立方值
    int cube[101];
    for (int i = 1; i <= N; i++) {
        cube[i] = i * i * i;
    }
    
    for (int a = 2; a <= N; a++) {
        for (int b = 2; b <= N; b++) {
            if (cube[b] > cube[a]) break;
            for (int c = b; c <= N; c++) {
                if (cube[b] + cube[c] > cube[a]) break;
                for (int d = c; d <= N; d++) {
                    if (cube[b] + cube[c] + cube[d] == cube[a]) {
                        printf("Cube = %d, Triple = (%d,%d,%d)\n", a, b, c, d);
                    } else if (cube[b] + cube[c] + cube[d] > cube[a]) {
                        break;
                    }
                }
            }
        }
    }
    
    return 0;
}
```

**优化效果**：减少了重复计算立方值的开销，代码更清晰。

### 扩展2：使用哈希表优化

可以使用哈希表存储立方值，将时间复杂度降低到 O(N³)：

```cpp
#include <iostream>
#include <unordered_map>
using namespace std;

int main() {
    int N;
    cin >> N;
    
    unordered_map<int, int> cube_map;
    for (int i = 2; i <= N; i++) {
        cube_map[i * i * i] = i;
    }
    
    for (int a = 2; a <= N; a++) {
        int a3 = a * a * a;
        for (int b = 2; b <= N; b++) {
            int b3 = b * b * b;
            if (b3 > a3) break;
            for (int c = b; c <= N; c++) {
                int c3 = c * c * c;
                int d3 = a3 - b3 - c3;
                if (d3 < c3) continue;  // d >= c，所以 d³ >= c³
                if (cube_map.count(d3)) {
                    int d = cube_map[d3];
                    if (d >= c && d <= N) {
                        cout << "Cube = " << a << ", Triple = (" << b << "," << c << "," << d << ")" << endl;
                    }
                }
            }
        }
    }
    
    return 0;
}
```

**优化效果**：将四层循环降为三层，通过哈希查找代替第四层枚举。

### 扩展3：使用二分查找优化

可以在第四层使用二分查找代替线性枚举：

```c
#include <stdio.h>

int main() {
    int N;
    scanf("%d", &N);
    
    int cube[101];
    for (int i = 1; i <= N; i++) {
        cube[i] = i * i * i;
    }
    
    for (int a = 2; a <= N; a++) {
        for (int b = 2; b <= N; b++) {
            if (cube[b] > cube[a]) break;
            for (int c = b; c <= N; c++) {
                if (cube[b] + cube[c] > cube[a]) break;
                
                // 使用二分查找找 d
                int target = cube[a] - cube[b] - cube[c];
                int left = c, right = N;
                while (left <= right) {
                    int mid = (left + right) / 2;
                    if (cube[mid] == target) {
                        printf("Cube = %d, Triple = (%d,%d,%d)\n", a, b, c, mid);
                        break;
                    } else if (cube[mid] < target) {
                        left = mid + 1;
                    } else {
                        right = mid - 1;
                    }
                }
            }
        }
    }
    
    return 0;
}
```

**优化效果**：第四层从 O(N) 降为 O(log N)，总体复杂度从 O(N⁴) 降为 O(N³ log N)。

### 扩展4：数学推导优化

根据数学推导，我们可以进一步缩小枚举范围：

```c
#include <stdio.h>

int main() {
    int N;
    scanf("%d", &N);
    
    for (int a = 2; a <= N; a++) {
        int a3 = a * a * a;
        
        // b 的上限：b³ < a³ → b < a
        for (int b = 2; b < a; b++) {
            int b3 = b * b * b;
            
            // c 的上限：b³ + c³ < a³ → c < (a³ - b³)^(1/3)
            int max_c = (int)(a3 - b3);
            // 粗略估计 c 的上限
            for (int c = b; c < a; c++) {
                int c3 = c * c * c;
                if (b3 + c3 > a3) break;
                
                int d3 = a3 - b3 - c3;
                // d 的范围：c <= d < a（因为 d³ = a³ - b³ - c³ < a³）
                for (int d = c; d < a; d++) {
                    if (d * d * d == d3) {
                        printf("Cube = %d, Triple = (%d,%d,%d)\n", a, b, c, d);
                        break;
                    }
                }
            }
        }
    }
    
    return 0;
}
```

**优化效果**：通过数学分析缩小枚举范围，b、c、d 都只需枚举到 a-1。

### 扩展5：寻找更多变量的完美立方

如果题目要求寻找五元组 (a, b, c, d, e) 满足 a³ = b³ + c³ + d³ + e³，可以增加一层循环：

```c
for (int a = 2; a <= N; a++) {

    for (int b = 2; b <= N; b++) {
        int b3 = b * b * b;
        if (b3 > a3) break;
        for (int c = b; c <= N; c++) {
            int c3 = c * c * c;
            if (b3 + c3 > a3) break;
            for (int d = c; d <= N; d++) {
                int d3 = d * d * d;
                if (b3 + c3 + d3 > a3) break;
                for (int e = d; e <= N; e++) {
                    int e3 = e * e * e;
                    if (b3 + c3 + d3 + e3 == a3) {
                        printf("Cube = %d, Triple = (%d,%d,%d,%d)\n", a, b, c, d, e);
                    } else if (b3 + c3 + d3 + e3 > a3) {
                        break;
                    }
                }
            }
        }
    }
}
```

### 扩展6：不同次方的完美等式

可以推广到更高次方，例如寻找满足 a⁴ = b⁴ + c⁴ + d⁴ 的四元组：

```c
#include <stdio.h>

int main() {
    int N;
    scanf("%d", &N);
    
    for (int a = 2; a <= N; a++) {
        int a4 = a * a * a * a;
        for (int b = 2; b <= N; b++) {
            int b4 = b * b * b * b;
            if (b4 > a4) break;
            for (int c = b; c <= N; c++) {
                int c4 = c * c * c * c;
                if (b4 + c4 > a4) break;
                for (int d = c; d <= N; d++) {
                    int d4 = d * d * d * d;
                    if (b4 + c4 + d4 == a4) {
                        printf("4th power = %d, Triple = (%d,%d,%d)\n", a, b, c, d);
                    } else if (b4 + c4 + d4 > a4) {
                        break;
                    }
                }
            }
        }
    }
    
    return 0;
}
```

### 扩展7：验证完美立方等式

```c
#include <stdio.h>

// 验证函数：检查四元组是否满足 a³ = b³ + c³ + d³
int verify(int a, int b, int c, int d) {
    return (a*a*a) == (b*b*b + c*c*c + d*d*d);
}

int main() {
    // 验证样例中的四元组
    printf("验证 (6,3,4,5): %s\n", verify(6,3,4,5) ? "成立" : "不成立");
```

## 数学原理补充

### 完美立方数的数学背景

完美立方等式 a³ = b³ + c³ + d³ 在数学上属于丢番图方程的一种。这类方程研究的是整数解的存在性和求解方法。

**著名的完美立方数：**
- 6³ = 3³ + 4³ + 5³ = 216（勾股数的扩展）
- 12³ = 6³ + 8³ + 10³ = 1728（6,8,10是3,4,5的倍数）
- 18³ = 2³ + 12³ + 16³ = 5832

**数学性质：**
1. 如果 (a, b, c, d) 是一个解，那么 (ka, kb, kc, kd) 也是一个解（k为整数）
2. 最小的完美立方数是6（因为 2³=8，无法表示为三个大于1的立方数之和）

### 优化策略对比

| 优化策略 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 |
|----------|------------|------------|------|------|
| 原始四层循环 | O(N⁴) | O(1) | 简单直观 | 效率最低 |
| 预计算立方值 | O(N⁴) | O(N) | 减少重复计算 | 空间略有增加 |
| 提前终止 | O(N⁴)~O(N³) | O(1) | 实际运行快 | 最差仍为O(N⁴) |
| 哈希表优化 | O(N³) | O(N) | 降为三层循环 | 需要额外空间 |
| 二分查找优化 | O(N³ log N) | O(N) | 第四层变为logN | 实现稍复杂 |
| 数学推导优化 | O(N³) | O(1) | 缩小枚举范围 | 需要数学分析 |

### 复杂度计算示例

对于 N = 100：
- 原始算法：100⁴ = 100,000,000 次循环
- 提前终止优化：约 1,000,000 次循环（减少约99%）
- 哈希表优化：100³ = 1,000,000 次循环
- 二分查找优化：100³ × log2(100) ≈ 700,000 次循环
