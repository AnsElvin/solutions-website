# 题目10：汉诺塔II

## 题目描述

有三根杆子A，B，C，A杆上有N个(N>1)穿孔圆盘，盘的尺寸由下到上依次变小。要求按下列规则将所有圆盘移至C杆：每次只能移动一个圆盘；大盘不能叠在小盘上面。提示：可将圆盘临时置于B杆，也可将从A杆移出的圆盘重新移回A杆，但都必须遵循上述两条规则。

问：如何移？最少要移动多少次？

### 输入

输入为一个整数后面跟三个单字符字符串。

整数为盘子的数目，后三个字符表示三个杆子的编号。

### 输出

输出每一步移动盘子的记录，一次移动一行。

每次移动的记录为例如`3:a->b`的形式，即把编号为3的盘子从a杆移至b杆。

我们约定圆盘从小到大编号为1,2,...,n，即最上面那个最小的圆盘编号为1，最下面最大的圆盘编号为n。

### 输入样例

```
3 a b c
```

### 输出样例

```
1:a->c
2:a->b
1:c->b
3:a->c
1:b->a
2:b->c
1:a->c
```

## 题目分析

这是汉诺塔I的变体，主要区别在于：
- 输入包含杆的编号（不再固定为A、B、C）
- 输出格式包含盘子编号

**关键信息提取：**
- 三根杆，编号由输入指定
- N个圆盘，编号1到n（1最小，n最大）
- 每次只能移动一个圆盘
- 大盘不能在小盘上面
- 输出格式：`编号:源杆->目标杆`

**考察要点：**
1. **递归思想**：与汉诺塔I相同
2. **输入处理**：读取杆的编号
3. **输出格式**：包含盘子编号

## 详细思路推演

### 第一步：理解问题

与汉诺塔I相同，只是杆的编号由输入指定，输出需要包含盘子编号。

### 第二步：递归结构

递归函数需要传递：
- n：盘子数量
- from：源杆
- via：中间杆
- to：目标杆
- 输出时需要知道当前移动的是哪个盘子

### 第三步：递归终止条件

当n=1时，直接移动盘子1。

### 第四步：递归过程

以n=3，杆为a,b,c为例：
```
Hanoi(3, a, b, c):
  Hanoi(2, a, c, b):
    Hanoi(1, a, b, c): 1:a->c
    2:a->b
    Hanoi(1, c, b, a): 1:c->b
  3:a->c
  Hanoi(2, b, a, c):
    Hanoi(1, b, c, a): 1:b->a
    2:b->c
    Hanoi(1, a, b, c): 1:a->c
```

## 分步算法讲解

### 算法步骤

1. **定义递归函数** `hanoi(n, from, via, to)`
2. **递归终止**：如果 n==1，输出 `1:from->to`，返回
3. **递归分解**：
   - 调用 hanoi(n-1, from, to, via)
   - 输出 `n:from->to`
   - 调用 hanoi(n-1, via, from, to)

## C语言代码

```c
#include <stdio.h>

void hanoi(int n, char from, char via, char to) {
    if (n == 1) {
        printf("%d:%c->%c\n", n, from, to);
        return;
    }
    hanoi(n - 1, from, to, via);
    printf("%d:%c->%c\n", n, from, to);
    hanoi(n - 1, via, from, to);
}

int main() {
    int n;
    char a, b, c;
    scanf("%d %c %c %c", &n, &a, &b, &c);
    hanoi(n, a, b, c);
    return 0;
}
```

## 常见错误与易错点

1. **输入格式错误**：输入是一个整数加三个字符，要用 `%c` 读取，注意空格。

2. **输出格式错误**：输出格式是 `编号:源杆->目标杆`，冒号和箭头不能少。

3. **递归终止条件**：没有 `if (n==1)` 会导致无限递归。

4. **参数顺序错误**：递归调用时三个杆的参数位置要正确。

5. **盘子编号**：输出的编号是当前移动的盘子编号，不是固定的1。

## 迭代实现

对于汉诺塔II，同样可以使用栈模拟递归过程，同时记录盘子编号：

```c
#include <stdio.h>
#include <stack>
using namespace std;

struct Step {
    int n;
    char from, via, to;
    bool visited;
};

void hanoi_iterative(int n, char from, char via, char to) {
    stack<Step> st;
    st.push({n, from, via, to, false});
    
    while (!st.empty()) {
        Step curr = st.top();
        st.pop();
        
        if (curr.n == 1) {
            printf("%d:%c->%c\n", curr.n, curr.from, curr.to);
            continue;
        }
        
        if (curr.visited) {
            printf("%d:%c->%c\n", curr.n, curr.from, curr.to);
        } else {
            st.push({curr.n - 1, curr.via, curr.from, curr.to, false});
            st.push({curr.n, curr.from, curr.via, curr.to, true});
            st.push({curr.n - 1, curr.from, curr.to, curr.via, false});
        }
    }
}

int main() {
    int n;
    char a, b, c;
    scanf("%d %c %c %c", &n, &a, &b, &c);
    hanoi_iterative(n, a, b, c);
    return 0;
}
```

**迭代实现原理：**
- 使用栈保存待处理的子问题
- `visited`标记表示该子问题是否已处理完前序步骤
- 输出时包含盘子编号`curr.n`

## 移动次数计算

汉诺塔II的移动次数与汉诺塔I相同，都是 2^n - 1 次。

可以单独计算移动次数：

```c
#include <stdio.h>

int main() {
    int n;
    char a, b, c;
    scanf("%d %c %c %c", &n, &a, &b, &c);
    
    int moves = (1 << n) - 1;  // 2^n - 1
    printf("移动次数: %d\n", moves);
    return 0;
}
```

**计算方法：**
- 使用位运算 `1 << n` 计算 2^n
- 减去1得到移动次数

## C语言基础知识补充

### 字符输入处理

在汉诺塔II中，需要读取三个字符作为杆的编号：

```c
char a, b, c;
scanf("%d %c %c %c", &n, &a, &b, &c);
```

**注意事项：**
- `%c`会读取任何字符，包括空格和换行符
- 在`%c`前加空格可以跳过空白字符
- 输入格式为：`数字 字符 字符 字符`，中间用空格分隔

### 字符变量

字符在C语言中本质上是整数（ASCII码）：

```c
char ch = 'A';  // 实际存储的是65（'A'的ASCII码）
printf("%d\n", ch);  // 输出65
printf("%c\n", ch);  // 输出A
```

常见字符的ASCII码：
- 'A'-'Z'：65-90
- 'a'-'z'：97-122
- '0'-'9'：48-57

### printf格式符

在输出时，需要使用正确的格式符：

```c
printf("%d:%c->%c\n", n, from, to);
```

| 格式符 | 说明 | 示例 |
|--------|------|------|
| `%d` | 十进制整数 | `printf("%d", 123)` → `123` |
| `%c` | 字符 | `printf("%c", 'A')` → `A` |
| `%s` | 字符串 | `printf("%s", "hello")` → `hello` |

### 变量作用域

在递归函数中，参数是局部变量：

```c
void hanoi(int n, char from, char via, char to) {
    // n, from, via, to 是每次调用的局部变量
    // 不同递归层次的参数值不同
}
```

每次递归调用时，参数值会被压入栈，返回时弹出恢复。

## 扩展思考

1. **迭代解法**：用栈模拟递归过程，避免栈溢出。

2. **输出移动次数**：可以计算 2^n - 1，不需要输出每一步。

3. **变种问题**：
   - 限制某些移动路径（如不能直接从A移到C）
   - 加入权重（不同盘子移动代价不同）
   - 多个源或目标柱子

4. **汉诺塔计数问题**：只统计移动次数，不输出具体步骤。

5. **逆向汉诺塔**：给定目标状态，求最少移动步数。

## 总结

汉诺塔II与汉诺塔I的核心算法相同，主要区别在于输入输出格式。

2. 输入读取三个杆的字符
3. 输出格式包含盘子编号
