# 题目09：汉诺塔I

## 题目描述

有三根杆子A，B，C，A杆上有N个(N>1)穿孔圆盘，盘的尺寸由下到上依次变小。要求按下列规则将所有圆盘移至C杆：每次只能移动一个圆盘；大盘不能叠在小盘上面。提示：可将圆盘临时置于B杆，也可将从A杆移出的圆盘重新移回A杆，但都必须遵循上述两条规则。

问：如何移？最少要移动多少次？

### 解法

解法的基本思想是递归。假设有A、B、C三个塔，A塔有N块盘，目标是把这些盘全部移到C塔。那么先把A塔顶部的N-1块盘移动到B塔，再把A塔剩下的大盘移到C，最后把B塔的N-1块盘移到C。每次移动多于一块盘时，则再次使用上述算法来移动。

### 输入

盘子个数n(1<=n<=20)

### 输出

输出移动的步骤，每行一步，如从A座移动到C座，输出"A->C"。

### 输入样例

```
3
```

### 输出样例

```
A->C
A->B
C->B
A->C
B->A
B->C
A->C
```

## 题目分析

这是经典的汉诺塔问题，是学习递归算法的经典案例。

**关键信息提取：**
- 三根杆：A、B、C
- N个圆盘，尺寸从下到上递减
- 每次只能移动一个圆盘
- 大盘不能在小盘上面
- 输出每一步移动

**考察要点：**
1. **递归思想**：如何将复杂问题分解为子问题
2. **递归终止条件**：n=1时直接移动
3. **递归调用**：n>1时分解为三个步骤

## 详细思路推演

### 第一步：理解问题

汉诺塔问题要求将N个圆盘从A杆移动到C杆，借助B杆，每次只能移动一个，且大盘不能在小盘下面。

### 第二步：分析递归结构

对于n个盘的问题：
- 如果n=1：直接 A->C
- 如果n>1：分三步
  1. 将n-1个盘从A移动到B（借助C）
  2. 将第n个盘从A移动到C
  3. 将n-1个盘从B移动到C（借助A）

### 第三步：递归终止条件

当n=1时，递归终止，直接移动。这是递归的出口。

### 第四步：递归过程

以n=3为例：
```
Hanoi(3, A, B, C):
  Hanoi(2, A, C, B):  // 将2个盘从A移到B
    Hanoi(1, A, B, C): A->C
    A->B
    Hanoi(1, C, B, A): C->B
  A->C
  Hanoi(2, B, A, C):  // 将2个盘从B移到C
    Hanoi(1, B, C, A): B->A
    Hanoi(1, A, B, C): A->C
    B->C
    Hanoi(1, A, C, B): A->C
```

### 第五步：分析时间复杂度

移动次数 = 2^n - 1
- n=1: 1次
- n=2: 3次
- n=3: 7次
- n=20: 1048575次

## 分步算法讲解

### 算法步骤

1. **定义递归函数** `hanoi(n, from, via, to)`
2. **递归终止**：如果 n==1，输出 from->to，返回
3. **递归分解**：
   - 调用 hanoi(n-1, from, to, via)
   - 输出 from->to
   - 调用 hanoi(n-1, via, from, to)

### 递归过程图（n=3）

```
初始状态：A=[3,2,1] B=[] C=[]

步骤1: A->C  A=[3,2] B=[] C=[1]
步骤2: A->B  A=[3] B=[2] C=[1]
步骤3: C->B  A=[3] B=[2,1] C=[]
步骤4: A->C  A=[] B=[2,1] C=[3]
步骤5: B->A  A=[1] B=[2] C=[3]
步骤6: B->C  A=[1] B=[] C=[3,2]
步骤7: A->C  A=[] B=[] C=[3,2,1]
```

## C语言代码

```c
#include <stdio.h>

void hanoi(int n, char from, char via, char to) {
    if (n == 1) {
        printf("%c->%c\n", from, to);
        return;
    }
    hanoi(n - 1, from, to, via);
    printf("%c->%c\n", from, to);
    hanoi(n - 1, via, from, to);
}

int main() {
    int n;
    scanf("%d", &n);
    hanoi(n, 'A', 'B', 'C');
    return 0;
}
```

## 常见错误与易错点

1. **忘记递归终止条件**：没有 `if (n==1)` 会导致无限递归，程序崩溃。

2. **参数顺序错误**：递归调用时三个参数的位置要正确，否则逻辑会乱。

3. **n的范围**：n最大为20，移动次数为 2^20-1 = 1048575，不会超时。

4. **递归深度**：递归深度为n，n最大20，不会栈溢出。

5. **输出格式错误**：注意是大写A、B、C，每步占一行。

## 复杂度分析

### 时间复杂度

**O(2^n)**

- 设T(n)为n个盘所需移动次数
- T(1) = 1
- T(n) = 2T(n-1) + 1
- 求解得 T(n) = 2^n - 1

### 空间复杂度

**O(n)**

- 递归调用栈的深度为n
- 每层递归占用常数空间

## 迭代实现

递归虽然直观，但对于较大的n会导致栈溢出。可以使用栈模拟递归过程：

```c
#include <stdio.h>
#include <stack>
using namespace std;

struct Step {
    int n;
    char from, via, to;
    bool visited;
};

void hanoi_iterative(int n, char from, char via, char to) {
    stack<Step> st;
    st.push({n, from, via, to, false});
    
    while (!st.empty()) {
        Step curr = st.top();
        st.pop();
        
        if (curr.n == 1) {
            printf("%c->%c\n", curr.from, curr.to);
            continue;
        }
        
        if (curr.visited) {
            printf("%c->%c\n", curr.from, curr.to);
        } else {
            st.push({curr.n - 1, curr.via, curr.from, curr.to, false});
            st.push({curr.n, curr.from, curr.via, curr.to, true});
            st.push({curr.n - 1, curr.from, curr.to, curr.via, false});
        }
    }
}

int main() {
    int n;
    scanf("%d", &n);
    hanoi_iterative(n, 'A', 'B', 'C');
    return 0;
}
```

**迭代实现原理：**
- 使用栈保存待处理的子问题
- `visited`标记表示该子问题是否已处理完前序步骤
- 第一次遇到子问题时，压入后续步骤（逆序压栈，保证正确顺序）
- 第二次遇到时（visited=true），执行当前步骤

## 数学证明

### 移动次数公式证明

**定理：** n个盘子的汉诺塔问题最少需要移动 2^n - 1 次。

**证明（数学归纳法）：**

1. **基础情况**：n=1时，需要1次移动。2^1 - 1 = 1，成立。

2. **归纳假设**：假设n=k时，最少需要2^k - 1次移动。

3. **归纳步骤**：n=k+1时：
   - 将k个盘子从A移到B：需要2^k - 1次
   - 将第k+1个盘子从A移到C：需要1次
   - 将k个盘子从B移到C：需要2^k - 1次
   - 总计：(2^k - 1) + 1 + (2^k - 1) = 2^(k+1) - 1

**结论**：由数学归纳法，公式成立。

### 递归深度分析

递归调用栈的深度为n，每次调用占用常数空间。

对于n=20，栈深度为20，不会溢出。
对于n=1000，栈深度为1000，可能导致栈溢出，需要使用迭代实现。

## C语言基础知识补充

### 递归函数的工作原理

递归函数的执行过程依赖于**调用栈**（Call Stack）：

1. **函数调用时**：将返回地址、参数、局部变量压入栈
2. **函数返回时**：弹出栈顶元素，恢复调用前的状态
3. **栈溢出**：递归深度过大时，栈空间耗尽

### 栈帧结构

```
栈底 ← 高地址
┌──────────────┐
│ main()       │
│   返回地址    │
│   参数        │
│   局部变量    │
├──────────────┤
│ hanoi(3)     │
│   返回地址    │
│   n=3        │
│   from='A'   │
│   via='B'    │
│   to='C'     │
├──────────────┤
│ hanoi(2)     │
│   返回地址    │
│   n=2        │
│   from='A'   │
│   via='C'    │
│   to='B'     │
├──────────────┤
│ hanoi(1)     │ ← 栈顶，执行中
│   返回地址    │
│   n=1        │
│   from='A'   │
│   via='B'    │
│   to='C'     │
└──────────────┘ ← 低地址
```

### 函数参数传递

在汉诺塔递归中，参数的位置非常重要：

```c
hanoi(n-1, from, to, via);   // 将n-1个盘从from移到via，借助to
printf("%c->%c", from, to);  // 将最大盘从from移到to
hanoi(n-1, via, from, to);   // 将n-1个盘从via移到to，借助from
```

参数顺序变化体现了角色的转换。

### 全局变量 vs 局部变量

在递归函数中，通常使用全局变量或参数传递状态：

- **全局变量**：不需要在递归调用中传递，代码简洁，但可能导致副作用
- **局部变量**：每次递归调用都有独立的副本，更安全，但需要额外的栈空间

## 扩展思考

1. **汉诺塔II**：增加一根杆子，递归解法有所不同，最少移动次数变为 O(2^sqrt(n))。

2. **迭代解法**：可以用栈模拟递归，避免递归调用，适用于n较大的情况。

3. **最少移动次数**：T(n) = 2^n - 1，可以通过数学公式直接计算。

4. **变种问题**：
   - 限制某些移动路径（如不能直接从A移到C）
   - 加入权重（不同盘子移动代价不同）
   - 多个源或目标柱子

5. **非递归最优解法**：对于4柱汉诺塔问题，Frame-Stewart算法可以减少移动次数。

## 总结

汉诺塔是经典的递归问题，核心思想是将大问题分解为小问题。

**解题要点：**
1. 明确递归终止条件 n=1
2. 三步分解：移n-1个到中间，移动最大盘，移n-1个到目标
3. 注意三个参数的位置关系
4. 时间复杂度 O(2^n)，空间复杂度 O(n)
5. 递归深度为n，对于较大n需要使用迭代实现

**学习价值：**
- 理解递归的核心思想（分治+回溯）
- 掌握递归终止条件的设计
- 理解调用栈的工作原理
- 学会将递归转换为迭代
