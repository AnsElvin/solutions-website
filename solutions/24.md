# 题目24：二进制密码锁

## 题目描述

在海拉鲁大陆有一种特殊的二进制密码锁，由 n 个相连的按钮组成（n < 30），按钮有凹/凸两种状态，用手按按钮会改变其状态。

按一个按钮时，跟它相邻的两个按钮状态也会反转。如果按的是最左或者最右边的按钮，该按钮只会影响到跟它相邻的一个按钮。

当前密码锁状态已知，需要解决的问题是，林克至少需要按多少次按钮，才能将密码锁转变为所期望的目标状态。

### 输入

两行，给出两个由 0、1 组成的等长字符串，表示当前/目标密码锁状态，其中 0 代表凹，1 代表凸。

### 输出

至少需要进行的按按钮操作次数，如果无法实现转变，则输出 impossible。

### 输入样例 1

```
011
000
```

### 输出样例 1

```
1
```

## 题目分析

这是一道经典的贪心算法题目，主要考察以下几个方面：

1. **贪心策略**：每一步做出最优选择，从左到右依次决定是否按下按钮
2. **状态传递**：按一个按钮会影响当前按钮及其相邻按钮的状态
3. **两种情况**：需要考虑是否按下第一个按钮的两种可能性
4. **模拟操作**：需要模拟按按钮后的状态变化

**关键信息提取：**
- 输入：两个等长的二进制字符串，长度 n < 30
- 输出：最少按键次数或 impossible
- 操作规则：按按钮 i，会反转按钮 i-1、i、i+1 的状态（边界按钮只反转两个）

## 详细思路推演

### 第一步：理解问题

题目要求找到最少按键次数，将当前状态转换为目标状态。例如：
- 当前：`011`，目标：`000`
- 按第 2 个按钮（中间的 1），会反转第 1、2、3 个按钮
- 结果：`011` → `000`，只需按 1 次

### 第二步：思考贪心策略

从左到右遍历每个按钮：
- 如果第 i 个按钮的状态与目标不同，必须按下第 i+1 个按钮
- 因为按第 i+1 个按钮会反转第 i 个按钮的状态
- 而按第 i 个按钮已经无法影响第 i-1 个按钮了

### 第三步：思考两种初始情况

第一个按钮是否按下会影响后续所有操作：
- 情况 1：不按第一个按钮
- 情况 2：按第一个按钮

需要分别计算两种情况的按键次数，取最小值。

### 第四步：思考状态表示

可以用数组来表示当前状态和目标状态的差异，或者直接模拟状态变化。

### 第五步：思考终止条件

遍历完所有按钮后，检查最后一个按钮的状态是否与目标一致：
- 如果一致，说明可以转换，记录按键次数
- 如果不一致，说明无法转换

## 分步算法讲解

### 算法步骤

1. **读取输入**：读取当前状态和目标状态字符串
2. **情况1**：不按第一个按钮，模拟贪心操作
3. **情况2**：按第一个按钮，模拟贪心操作
4. **比较结果**：取两种情况中的最小值，如果都无法转换则输出 impossible

### 核心贪心逻辑

```
for i from 0 to n-2:
    if 当前状态[i] != 目标状态[i]:
        按下按钮 i+1
        反转 当前状态[i]、当前状态[i+1]、当前状态[i+2]
        按键次数++
```

### 流程图

```
开始
  ↓
读取 current 和 target
  ↓
计算情况1（不按第一个按钮）
  ┌─────────────────────────────┐
  │ temp = current 副本        │
  │ count1 = 0                 │
  │ for i = 0 to n-2:         │
  │     if temp[i] != target[i]:│
  │         count1++           │
  │         反转 temp[i]       │
  │         反转 temp[i+1]     │
  │         反转 temp[i+2] (如果存在) │
  │ if temp[n-1] == target[n-1]: │
  │     valid1 = true          │
  └─────────────────────────────┘
  ↓
计算情况2（按第一个按钮）
  ┌─────────────────────────────┐
  │ temp = current 副本        │
  │ count2 = 1                 │
  │ 反转 temp[0]               │
  │ 反转 temp[1]               │
  │ for i = 0 to n-2:         │
  │     if temp[i] != target[i]:│
  │         count2++           │
  │         反转 temp[i]       │

  ┌─────────────────────────────┐

  │ else if valid1:            │

  │ else if valid2:            │
  │     result = count2        │
  │ else:                      │

  ↓

  ↓

```

### 算法示例（输入：011 → 000）

**情况1（不按第一个按钮）：**
```
初始: temp = [0, 1, 1], target = [0, 0, 0]

i = 0: temp[0] = 0 == target[0] = 0, 不操作

i = 1: temp[1] = 1 != target[1] = 0, 按下按钮 2
  反转 temp[1], temp[2]
  temp = [0, 0, 0], count1 = 1

检查 temp[2] = 0 == target[2] = 0, valid1 = true
```

**情况2（按第一个按钮）：**
```
初始: temp = [0, 1, 1], target = [0, 0, 0]

按下按钮 0: 反转 temp[0], temp[1]
  temp = [1, 0, 1], count2 = 1

i = 0: temp[0] = 1 != target[0] = 0, 按下按钮 1
  反转 temp[0], temp[1], temp[2]
  temp = [0, 1, 0], count2 = 2

i = 1: temp[1] = 1 != target[1] = 0, 按下按钮 2
  反转 temp[1], temp[2]
  temp = [0, 0, 1], count2 = 3

检查 temp[2] = 1 != target[2] = 0, valid2 = false
```

**结果：** min(1, 不可能) = 1

## C语言代码

```c
#include <stdio.h>
#include <string.h>
#include <stdlib.h>

int n;
char current[35], target[35];

int solve(int press_first) {
    char temp[35];
    strcpy(temp, current);
    int count = 0;
    
    if (press_first) {
        count++;
        temp[0] ^= 1;
        if (n > 1) temp[1] ^= 1;
    }
    
    for (int i = 0; i < n - 1; i++) {
        if (temp[i] != target[i]) {
            count++;
            temp[i] ^= 1;
            temp[i + 1] ^= 1;
            if (i + 2 < n) temp[i + 2] ^= 1;
        }
    }
    
    if (temp[n - 1] == target[n - 1]) {
        return count;
    }
    return -1;
}

int main() {
    scanf("%s%s", current, target);
    n = strlen(current);
    
    int res1 = solve(0);
    int res2 = solve(1);
    
    int ans = -1;
    if (res1 != -1 && res2 != -1) {
        ans = res1 < res2 ? res1 : res2;
    } else if (res1 != -1) {
        ans = res1;
    } else if (res2 != -1) {
        ans = res2;
    }
    
    if (ans == -1) {
        printf("impossible\n");
    } else {
        printf("%d\n", ans);
    }
    
    return 0;
}
```

## C++代码

```cpp
#include <iostream>
#include <string>
#include <algorithm>
using namespace std;

int solve(string s, const string& target) {
    int n = s.size();
    int count = 0;
    
    for (int i = 0; i < n - 1; i++) {
        if (s[i] != target[i]) {
            count++;
            s[i] ^= 1;
            s[i + 1] ^= 1;
            if (i + 2 < n) s[i + 2] ^= 1;
        }
    }
    
    return s.back() == target.back() ? count : -1;
}

int main() {
    string current, target;
    cin >> current >> target;
    
    int res1 = solve(current, target);
    
    string current2 = current;
    current2[0] ^= 1;
    if (current2.size() > 1) current2[1] ^= 1;
    int res2 = solve(current2, target);
    res2 = res2 == -1 ? -1 : res2 + 1;
    
    int ans = -1;
    if (res1 != -1 && res2 != -1) {
        ans = min(res1, res2);
    } else if (res1 != -1) {
        ans = res1;
    } else if (res2 != -1) {
        ans = res2;
    }
    
    if (ans == -1) {
        cout << "impossible" << endl;
    } else {
        cout << ans << endl;
    }
    
    return 0;
}
```

## 常见错误与易错点

1. **忘记考虑两种情况**：只计算不按第一个按钮的情况，可能漏掉最优解。

2. **边界处理错误**：按下最后一个按钮时，不需要反转不存在的 i+2 位。

3. **字符串索引错误**：C/C++ 字符串索引从 0 开始，注意不要越界。

4. **状态复制问题**：在递归或多次计算时，必须使用副本而不是直接修改原字符串。

5. **反转操作错误**：按按钮 i 时，应该反转 i-1、i、i+1，而不是 i、i+1、i+2。

6. **字符类型问题**：字符串中的字符是 '0' 和 '1'，反转时使用 `^= 1` 会将 '0' 变为 '1'（因为 '0' = 48，'1' = 49，48 ^ 1 = 49）。

7. **结果比较错误**：需要正确处理两种情况都不可能、只有一种可能、两种都可能的情况。

8. **输入读取问题**：使用 `scanf("%s%s")` 或 `cin` 读取字符串时，确保输入格式正确。

## 复杂度分析

### 时间复杂度

**O(n)**

- 两种情况各遍历一次字符串：O(n) + O(n) = O(n)
- 每个字符最多访问两次
- 总时间复杂度为 O(n)

### 空间复杂度

**O(n)**

- 需要存储当前状态和目标状态的副本
- 字符串长度为 n
- 总空间复杂度为 O(n)

## 扩展思考

### 扩展1：三进制密码锁

如果按钮有三种状态（0、1、2），每次按下循环切换状态：

```cpp
#include <iostream>
#include <string>
#include <algorithm>
using namespace std;

int solve(string s, const string& target) {
    int n = s.size();
    int count = 0;
    
    for (int i = 0; i < n - 1; i++) {
        int diff = (target[i] - s[i] + 3) % 3;
        if (diff != 0) {
            count += diff;
            for (int j = 0; j < diff; j++) {
                s[i] = (s[i] - '0' + 1) % 3 + '0';
                s[i + 1] = (s[i + 1] - '0' + 1) % 3 + '0';
                if (i + 2 < n) s[i + 2] = (s[i + 2] - '0' + 1) % 3 + '0';
            }
        }
    }
    
    int diff = (target.back() - s.back() + 3) % 3;
    return diff == 0 ? count : -1;
}

int main() {
    string current, target;
    cin >> current >> target;
    
    int res1 = solve(current, target);
    
    string current2 = current;
    current2[0] = (current2[0] - '0' + 1) % 3 + '0';
    if (current2.size() > 1) current2[1] = (current2[1] - '0' + 1) % 3 + '0';
    int res2 = solve(current2, target);
    res2 = res2 == -1 ? -1 : res2 + 1;
    
    int ans = -1;
    if (res1 != -1 && res2 != -1) {
        ans = min(res1, res2);
    } else if (res1 != -1) {
        ans = res1;
    } else if (res2 != -1) {
        ans = res2;
    }
    
    if (ans == -1) {
        cout << "impossible" << endl;
    } else {
        cout << ans << endl;
    }
    
    return 0;
}
```

### 扩展2：BFS 解法

对于 n < 30，可以使用 BFS 找到最短路径：

```cpp
#include <iostream>
#include <string>
#include <queue>
#include <unordered_set>
using namespace std;

int main() {
    string start, target;
    cin >> start >> target;
    
    if (start == target) {
        cout << 0 << endl;
        return 0;
    }
    
    queue<pair<string, int>> q;
    unordered_set<string> visited;
    
    q.push({start, 0});
    visited.insert(start);
    
    while (!q.empty()) {
        auto [state, steps] = q.front();
        q.pop();
        
        for (int i = 0; i < state.size(); i++) {
            string next_state = state;
            next_state[i] ^= 1;
            if (i > 0) next_state[i - 1] ^= 1;
            if (i < state.size() - 1) next_state[i + 1] ^= 1;
            
            if (next_state == target) {
                cout << steps + 1 << endl;
                return 0;
            }
            
            if (!visited.count(next_state)) {
                visited.insert(next_state);
                q.push({next_state, steps + 1});
            }
        }
    }
    
    cout << "impossible" << endl;
    return 0;
}
```

**注意**：BFS 解法的时间复杂度为 O(2^n)，当 n = 30 时无法运行。贪心算法更高效。

### 扩展3：双向 BFS

可以使用双向 BFS 从起点和终点同时搜索，减少搜索空间：

```cpp
#include <iostream>
#include <string>
#include <queue>
#include <unordered_map>
using namespace std;

int main() {
    string start, target;
    cin >> start >> target;
    
    if (start == target) {
        cout << 0 << endl;
        return 0;
    }
    
    queue<string> q1, q2;
    unordered_map<string, int> dist1, dist2;
    
    q1.push(start);
    dist1[start] = 0;
    
    q2.push(target);
    dist2[target] = 0;
    
    while (!q1.empty() && !q2.empty()) {
        string state1 = q1.front();
        q1.pop();
        
        for (int i = 0; i < state1.size(); i++) {
            string next_state = state1;
            next_state[i] ^= 1;
            if (i > 0) next_state[i - 1] ^= 1;
            if (i < state1.size() - 1) next_state[i + 1] ^= 1;
            
            if (dist2.count(next_state)) {
                cout << dist1[state1] + 1 + dist2[next_state] << endl;
                return 0;
            }
            
            if (!dist1.count(next_state)) {
                dist1[next_state] = dist1[state1] + 1;
                q1.push(next_state);
            }
        }
        
        string state2 = q2.front();
        q2.pop();
        
        for (int i = 0; i < state2.size(); i++) {
            string next_state = state2;
            next_state[i] ^= 1;
            if (i > 0) next_state[i - 1] ^= 1;
            if (i < state2.size() - 1) next_state[i + 1] ^= 1;
            
            if (dist1.count(next_state)) {
                cout << dist2[state2] + 1 + dist1[next_state] << endl;
                return 0;
            }
            
            if (!dist2.count(next_state)) {
                dist2[next_state] = dist2[state2] + 1;
                q2.push(next_state);
            }
        }
    }
    
    cout << "impossible" << endl;
    return 0;
}
```

## 总结

这道题是贪心算法的经典应用，核心在于从左到右依次决定是否按下按钮。

**核心要点：**

1. **贪心策略**：如果第 i 位不匹配，必须按下第 i+1 位来修正
2. **两种情况**：需要考虑是否按下第一个按钮的两种可能性
3. **边界处理**：按下边界按钮时，只影响相邻的一个按钮
4. **最终检查**：遍历结束后，必须检查最后一位是否匹配

**学习收获：**

1. 理解了贪心算法的应用场景
2. 掌握了状态模拟的技巧
3. 学会了处理边界情况
4. 理解了为什么贪心算法在这里是最优的

这道题的解法简洁高效，时间复杂度为 O(n)，是处理此类问题的最优方法。
