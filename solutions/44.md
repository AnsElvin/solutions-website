# 题目44：求二进制中1的个数

## 题目分析

| 字段 | 内容 |
|------|------|
| **题目来源** | 经典位运算题 |
| **核心考点** | 位运算、lowbit操作、补码表示 |
| **数据范围** | 32位整数（-2^31 ~ 2^31 - 1） |
| **时间限制** | O(1) |
| **空间限制** | O(1) |

**题目描述**：输入一个32位整数，输出该数二进制表示中1的个数。负数在计算机中用其绝对值的补码来表示。

**示例**：
- 输入：9，二进制：1001，输出：2
- 输入：-2，二进制补码：11111111111111111111111111111110，输出：31

---

## 详细思路推演

### 思路1：逐位检查

**步骤1**：遍历每一位

对于32位整数，逐位检查每一位是否为1：

```
count = 0
for i from 0 to 31:
    if (n & (1 << i)) != 0:
        count++
return count
```

**步骤2**：示例分析

对于n=9：
- 二进制：1001
- 第0位：9 & 1 = 1，count=1
- 第1位：9 & 2 = 0，不变
- 第2位：9 & 4 = 0，不变
- 第3位：9 & 8 = 8，count=2
- 其他位：0
- 返回2

对于n=-2（补码表示）：
- 二进制补码：11111111111111111111111111111110
- 除了第0位是0，其余31位都是1
- 返回31

---

### 思路2：lowbit操作

**步骤1**：理解lowbit

`lowbit(n)` 返回n的二进制表示中最低位1及其后面的所有0组成的数：
- `lowbit(n) = n & (-n)`
- 原理：负数的补码 = 反码 + 1

**步骤2**：使用lowbit统计1的个数

每次消除最低位的1，直到n变为0：

```
count = 0
while n != 0:
    count++
    n -= lowbit(n)
return count
```

**步骤3**：示例分析

对于n=9（二进制1001）：
- lowbit(9) = 1，n = 9 - 1 = 8，count=1
- lowbit(8) = 8，n = 8 - 8 = 0，count=2
- 返回2

对于n=-2：
- lowbit(-2) = 2，n = -2 - 2 = -4，count=1
- lowbit(-4) = 4，n = -4 - 4 = -8，count=2
- ...继续直到n=-2^31
- lowbit(-2^31) = 2^31，n = -2^31 - 2^31 = 0，count=31
- 返回31

---

### 思路3：分治算法

**步骤1**：分治思想

将32位数分成两半，每半16位，统计每半中1的个数，然后相加：

```
n = (n & 0x55555555) + ((n >> 1) & 0x55555555)  // 每2位统计
n = (n & 0x33333333) + ((n >> 2) & 0x33333333)  // 每4位统计
n = (n & 0x0f0f0f0f) + ((n >> 4) & 0x0f0f0f0f)  // 每8位统计
n = (n & 0x00ff00ff) + ((n >> 8) & 0x00ff00ff)  // 每16位统计
n = (n & 0x0000ffff) + ((n >> 16) & 0x0000ffff) // 每32位统计
return n
```

**步骤2**：原理

- 0x55555555 = 01010101 01010101 01010101 01010101
- 0x33333333 = 00110011 00110011 00110011 00110011
- 0x0f0f0f0f = 00001111 00001111 00001111 00001111
- 0x00ff00ff = 00000000 11111111 00000000 11111111
- 0x0000ffff = 00000000 00000000 11111111 11111111

每次操作将相邻位的1的个数相加，最终得到总的1的个数。

## 分步算法讲解

### 算法1：逐位检查

```
count_ones(n):
    count = 0
    for i = 0 to 31:
        if (n & (1 << i)) != 0:
            count++
    return count
```

**时间复杂度**：O(32) = O(1)
**空间复杂度**：O(1)

### 算法2：lowbit操作

```
count_ones(n):
    count = 0
    while n != 0:
        count++
        n &= n - 1  // 等价于 n -= lowbit(n)
    return count
```

**时间复杂度**：O(k)，k为1的个数，最坏O(32) = O(1)
**空间复杂度**：O(1)

**原理**：`n & (n - 1)` 会消除n的二进制表示中最低位的1

### 算法3：分治算法

```
count_ones(n):
    n = (n & 0x55555555) + ((n >> 1) & 0x55555555)
    n = (n & 0x33333333) + ((n >> 2) & 0x33333333)
    n = (n & 0x0f0f0f0f) + ((n >> 4) & 0x0f0f0f0f)
    n = (n & 0x00ff00ff) + ((n >> 8) & 0x00ff00ff)
    n = (n & 0x0000ffff) + ((n >> 16) & 0x0000ffff)
    return n
```

**时间复杂度**：O(1)，固定5次操作
**空间复杂度**：O(1)

## C语言代码

### 方法1：逐位检查

```c
#include <stdio.h>

int count_ones(int n) {
    int count = 0;
    for (int i = 0; i < 32; i++) {
        if ((n & (1 << i)) != 0) {
            count++;
        }
    }
    return count;
}

int main() {
    int n;
    while (scanf("%d", &n) == 1) {
        printf("%d\n", count_ones(n));
    }
    return 0;
}
```

**代码解释**：
- 使用`1 << i`生成第i位为1的掩码
- `n & (1 << i)`判断第i位是否为1

### 方法2：lowbit操作

```c
#include <stdio.h>

int count_ones(int n) {
    int count = 0;
    while (n != 0) {
        count++;
        n &= n - 1;
    }
    return count;
}

int main() {
    int n;
    while (scanf("%d", &n) == 1) {
        printf("%d\n", count_ones(n));
    }
    return 0;
}
```

**代码解释**：
- `n & (n - 1)`消除最低位的1
- 循环直到n变为0，统计消除次数

### 方法3：分治算法

```c
#include <stdio.h>

int count_ones(int n) {
    n = (n & 0x55555555) + ((n >> 1) & 0x55555555);
    n = (n & 0x33333333) + ((n >> 2) & 0x33333333);
    n = (n & 0x0f0f0f0f) + ((n >> 4) & 0x0f0f0f0f);
    n = (n & 0x00ff00ff) + ((n >> 8) & 0x00ff00ff);
    n = (n & 0x0000ffff) + ((n >> 16) & 0x0000ffff);
    return n;
}

int main() {
    int n;
    while (scanf("%d", &n) == 1) {
        printf("%d\n", count_ones(n));
    }
    return 0;
}
```

**代码解释**：
- 使用分治思想，逐级统计相邻位的1的个数
- 最终结果存储在n的低32位中

## 常见错误与易错点

| 错误类型 | 错误描述 | 错误示例 | 修正方法 |
|----------|----------|----------|----------|
| **负数处理错误** | 未考虑负数的补码表示 | 使用`n > 0`判断 | 直接处理，C语言自动使用补码 |
| **位移溢出** | 循环条件错误 | `for (int i = 0; n; i++)` | `for (int i = 0; i < 32; i++)` |
| **逻辑错误** | 条件判断错误 | `if (n & (1 << i))` | `if ((n & (1 << i)) != 0)` |
| **lowbit理解错误** | 错误使用lowbit | `n &= -n` | `n &= n - 1` |
| **分治掩码错误** | 掩码值错误 | `0xaaaaaaa` | `0x55555555` |
| **变量类型错误** | 使用unsigned int | `unsigned int n` | 使用`signed int n` |
| **输入输出错误** | 未处理多组输入 | 只读一次 | `while (scanf("%d", &n) == 1)` |
| **位运算优先级错误** | 未加括号 | `n & 1 << i` | `n & (1 << i)` |

## 扩展思考

### 思考1：为什么lowbit操作有效

`n & (n - 1)` 的原理：
- n-1会将n的最低位1变为0，后面的所有0变为1
- 与n进行&操作后，最低位的1被消除

例如：
- n = 9 = 1001
- n-1 = 8 = 1000
- n & (n-1) = 1000 = 8

### 思考2：分治算法的原理

分治算法的每一步：
1. 每2位统计：将相邻2位的1的个数相加
2. 每4位统计：将相邻4位的1的个数相加
3. 每8位统计：将相邻8位的1的个数相加
4. 每16位统计：将相邻16位的1的个数相加
5. 每32位统计：将相邻32位的1的个数相加

最终得到总的1的个数。

### 思考3：内置函数

在实际编程中，可以使用内置函数：
- GCC：`__builtin_popcount(n)`
- MSVC：`_mm_popcnt_u32(n)`

但题目要求手动实现，所以需要自己编写代码。

### 思考4：64位整数

如果扩展到64位整数，需要：
- 逐位检查：循环64次
- lowbit操作：最坏64次循环
- 分治算法：增加一步64位统计

## 总结

求二进制中1的个数是一个经典的位运算问题。

**核心要点**：
- 理解二进制表示和补码原理
- 掌握位运算技巧：与、或、异或、位移
- 理解lowbit操作的原理
- 分治算法的巧妙应用

**学习收获**：
- 理解位运算的基本操作
- 掌握统计二进制中1的个数的多种方法
- 学会处理负数的补码表示
- 了解分治算法在位运算中的应用

**关键技巧**：
- `n & (n - 1)` 消除最低位的1
- `n & (-n)` 获取最低位的1（lowbit）
- 分治算法可以在常数时间内完成统计

这个问题体现了位运算的精妙之处，通过不同的方法可以在O(1)时间内完成计算。
