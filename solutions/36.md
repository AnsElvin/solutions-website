# 题目36：数的三次方根

## 题目分析

| 字段 | 内容 |
|------|------|
| **题目来源** | 经典二分法题 |
| **核心考点** | 二分法、浮点精度控制、负数处理 |
| **数据范围** | -10000 ≤ n ≤ 10000 |
| **精度要求** | 保留6位小数 |
| **输入** | 一个浮点数n |
| **输出** | n的三次方根 |

**题目描述**：给定一个浮点数n，求它的三次方根，结果保留6位小数。

---

## 详细思路推演

### 思路：二分法求三次方根

**步骤1：理解三次方根的数学性质**

对于任意实数n，其三次方根x满足 x³ = n。

三次函数 f(x) = x³ - n 的性质：
- 当 n > 0 时，根在 (0, max(n, 1)) 之间
- 当 n = 0 时，根为 0
- 当 n < 0 时，根在 (min(n, -1), 0) 之间

**步骤2：确定二分区间**

由于 n ∈ [-10000, 10000]，可以统一设置区间：
- left = -100
- right = 100

这个区间足够覆盖所有可能的根。

**步骤3：二分法迭代过程**

```
输入 n = 1000.0
初始区间: [-100, 100]
迭代1: mid = 0, f(0) = -1000 < 0, 根在(0, 100)
迭代2: mid = 50, f(50) = 124000 > 0, 根在(0, 50)
迭代3: mid = 25, f(25) = 15625 - 1000 = 14625 > 0, 根在(0, 25)
迭代4: mid = 12.5, f(12.5) ≈ 1953 - 1000 = 953 > 0, 根在(0, 12.5)
迭代5: mid = 6.25, f(6.25) ≈ 244 - 1000 = -756 < 0, 根在(6.25, 12.5)
...
继续迭代直到区间长度小于 1e-7
```

**步骤4：精度控制**

当区间长度 right - left < 1e-7 时，认为达到精度要求（保留6位小数需要至少7位精度）。

**步骤5：处理负数**

三次方根可以处理负数，因为 (-x)³ = -x³，所以：
- 如果 n < 0，先求 |n| 的三次方根，再取负

## 分步算法讲解

### 算法步骤

```
cube_root(n):
    left = -100.0
    right = 100.0
    precision = 1e-7
    
    while right - left > precision:
        mid = (left + right) / 2
        if mid * mid * mid < n:
            left = mid
        else:
            right = mid
    
    return (left + right) / 2

输入 n
输出 cube_root(n) 保留6位小数
```

## C语言代码

```c
#include <stdio.h>

double cube_root(double n) {
    double left = -100.0, right = 100.0;
    double precision = 1e-7;
    
    while (right - left > precision) {
        double mid = (left + right) / 2;
        if (mid * mid * mid < n) {
            left = mid;
        } else {
            right = mid;
        }
    }
    
    return (left + right) / 2;
}

int main() {
    double n;
    scanf("%lf", &n);
    
    double result = cube_root(n);
    printf("%.6f\n", result);
    
    return 0;
}
```

**代码解释**：

| 函数 | 功能说明 |
|------|----------|
| `cube_root(n)` | 使用二分法求解n的三次方根 |
| `main` | 读取输入，调用函数，输出结果 |

**关键点**：
- 使用 `double` 类型保证精度
- 区间设置为 [-100, 100]，覆盖所有可能的根
- 精度设置为 1e-7，确保输出6位小数准确
- 输出格式 `%.6f` 保证6位小数

## 常见错误与易错点

| 错误类型 | 错误描述 | 错误示例 | 修正方法 |
|----------|----------|----------|----------|
| **区间选择错误** | 区间没有覆盖负数 | `left=0, right=100` | `left=-100, right=100` |
| **精度不够** | 精度设置过大 | `precision = 1e-5` | `precision = 1e-7` |
| **浮点精度问题** | 使用 float 类型精度不够 | `float left = -100.0` | 使用 `double` 类型 |
| **输出格式错误** | 输出小数位数不足 | `printf("%f", result)` | `printf("%.6f", result)` |
| **循环条件错误** | 使用 `>=` 而非 `>` | `while (right - left >= precision)` | `while (right - left > precision)` |
| **负数处理错误** | 认为三次方根只适用于正数 | 直接返回0或报错 | 二分法天然支持负数 |
| **特殊情况未处理** | n=0时未特殊处理 | 仍进行迭代 | 二分法可以处理，无需特殊处理 |
| **溢出问题** | mid³计算溢出 | `mid * mid * mid` 当mid很大时 | 本题数据范围不会溢出 |

## 扩展思考

### 思考1：牛顿迭代法求三次方根

```cpp
double cube_root_newton(double n) {
    double x = 1.0;
    for (int i = 0; i < 50; i++) {
        x = x - (x * x * x - n) / (3 * x * x);
        if (fabs(x * x * x - n) < 1e-7) break;
    }
    return x;
}
```

牛顿迭代法收敛更快，通常只需要5-10次迭代。

### 思考2：一般n次方根

可以扩展为求n次方根：
- 二分法：判断 mid^n 与目标值的大小
- 牛顿法：x = x - (x^n - target) / (n * x^(n-1))

### 思考3：对比不同方法

| 方法 | 优点 | 缺点 |
|------|------|------|
| 二分法 | 简单可靠，保证收敛 | 收敛速度较慢 |
| 牛顿迭代法 | 收敛速度快 | 需要计算导数，初始值敏感 |
| 数学公式 | 直接计算 | 精度有限，可能有数值问题 |

## 总结

数的三次方根是一个经典的数值计算问题，使用二分法可以可靠地求解。

**核心要点**：
- 二分法通过不断缩小区间来逼近三次方根
- 设置合适的初始区间，覆盖所有可能的输入
- 使用 `double` 类型保证精度
- 控制循环条件确保达到精度要求
- 二分法天然支持负数输入

**学习收获**：
- 理解二分法在数值计算中的应用
- 掌握浮点精度控制技巧
- 了解牛顿迭代法的原理
- 学会处理特殊情况（负数、零）

**对比二分法和牛顿法**：

| 特性 | 二分法 | 牛顿迭代法 |
|------|--------|------------|
| 收敛速度 | 线性收敛 | 二次收敛 |
| 需要导数 | 否 | 是 |
| 实现难度 | 简单 | 中等 |
| 收敛保证 | 保证收敛 | 初始值合适时收敛 |
| 适用范围 | 所有实数 | 所有实数 |

二分法是最稳妥的选择，而牛顿法在初始值合适时效率更高。
