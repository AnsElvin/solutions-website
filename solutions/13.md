# 题目13：输出N皇后的全部摆法

## 题目描述

随着编程水平的进阶，小华知道是全面扩充小鲁知识面，帮助他快速升级的时候到了，他再给小鲁布置了道经典题：

输入一个正整数N，请写一个程序，输出N皇后问题的全部摆法。

### 输入

输入皇后的个数n(n<=13)

### 输出

输出长度为n的正整数。

输出结果里的每一行都代表一种摆法。

行里的第i个数字如果是n，就代表第i行的皇后应该放在第n列。

皇后的行、列编号都是从1开始算。

### 输入样例

```
4
```

### 输出样例

```
2413
3142
```

## 题目分析

N皇后问题是经典的回溯算法问题，要求在n×n的棋盘上放置n个皇后，使得任意两个皇后不在同一行、同一列、同一对角线上。

**关键信息提取：**
- 棋盘大小：n×n
- 放置n个皇后
- 约束条件：不同行、不同列、不同对角线
- 输出格式：每行一个数字序列，表示每行皇后所在的列号
- n<=13

**考察要点：**
1. **回溯算法**：DFS+剪枝
2. **约束条件检查**：如何判断冲突
3. **对角线判断**：如何判断是否在同一对角线

## 详细思路推演

### 第一步：理解问题

我们需要在n×n的棋盘上放置n个皇后，使得任意两个皇后不冲突。每行只能放一个皇后（因为不同行），所以问题转化为为每行选择一个列。

### 第二步：约束条件

对于第row行选择第col列：
- 列冲突：其他行已经有皇后在col列
- 对角线冲突：|row - other_row| == |col - other_col|

### 第三步：DFS思想

与全排列类似，但增加了约束条件检查。

```
dfs(row):
  if row == n:
    输出当前摆法
    return
  for col = 1 to n:
    if col未被使用且对角线无冲突:
      使用col，记录到path[row]
      dfs(row+1)
      取消使用col
```

### 第四步：冲突检测

使用三个数组记录：
- `col_used[]`：列是否被使用
- `diag1[]`：主对角线是否被使用（row-col+n）
- `diag2[]`：副对角线是否被使用（row+col）

### 第五步：输出格式

每行输出n个数字，第i个数字表示第i行皇后所在的列号。

## 分步算法讲解

### 算法步骤

1. **初始化**：创建col_used、diag1、diag2数组和path数组
2. **DFS函数**：参数为当前行row
3. **递归终止**：row == n时输出并返回
4. **遍历列**：for col=1 to n
5. **检查冲突**：col_used[col] && diag1[row-col+n] && diag2[row+col]
6. **选择**：标记使用，记录path[row]=col
7. **递归**：dfs(row+1)
8. **回溯**：取消标记

### N=4时的搜索过程

```
row=1:
  col=1: diag1[1-1+4]=diag1[4], diag2[1+1]=diag2[2] → 放置
    row=2:
      col=1: 列冲突
      col=2: 对角线冲突(1-2=1-2)
      col=3: diag1[2-3+4]=diag1[3], diag2[2+3]=diag2[5] → 放置
        row=3:
          col=1: diag1[3-1+4]=diag1[6], diag2[3+1]=diag2[4] → 放置
            row=4: 所有列都冲突 → 回溯
          col=2: 列冲突
          col=3: 列冲突
          col=4: 对角线冲突 → 回溯
      col=4: diag1[2-4+4]=diag1[2], diag2[2+4]=diag2[6] → 放置
        row=3:
          col=1: 对角线冲突
          col=2: diag1[3-2+4]=diag1[5], diag2[3+2]=diag2[5] → 放置
            row=4:
              col=1: 对角线冲突
              col=2: 列冲突
              col=3: diag1[4-3+4]=diag1[5], diag2[4+3]=diag2[7] → 放置
                row=5 → 输出 2413
              col=4: 列冲突
  col=2: ...继续搜索，最终找到 3142
```

## C语言代码

```c
#include <stdio.h>

int n;
int path[20];
int col_used[20];
int diag1[40];
int diag2[40];

void dfs(int row) {
    if (row == n + 1) {
        for (int i = 1; i <= n; i++) {
            printf("%d", path[i]);
        }
        printf("\n");
        return;
    }
    for (int col = 1; col <= n; col++) {
        if (!col_used[col] && !diag1[row - col + n] && !diag2[row + col]) {
            col_used[col] = 1;
            diag1[row - col + n] = 1;
            diag2[row + col] = 1;
            path[row] = col;
            dfs(row + 1);
            col_used[col] = 0;
            diag1[row - col + n] = 0;
            diag2[row + col] = 0;
        }
    }
}

int main() {
    scanf("%d", &n);
    dfs(1);
    return 0;
}
```

## 常见错误与易错点

1. **行号从1开始**：题目要求行、列编号从1开始，不是从0开始。

2. **对角线冲突判断**：`|row - other_row| == |col - other_col|`，或者使用diag1和diag2数组。

3. **diag1数组下标**：`row - col`可能为负数，需要加n偏移。

4. **递归终止条件**：`row == n + 1`而不是`row == n`，因为行从1开始。

5. **数组大小**：diag1和diag2数组需要足够大，至少2n+1。

## 复杂度分析

### 时间复杂度

**O(n!)**（最坏情况）

- 第一行：n种选择
- 第二行：最多n-2种选择（排除同一列和对角线）
- 第三行：最多n-4种选择
- 总体：O(n!)

### 空间复杂度

**O(n)**

- path数组：O(n)
- 三个标记数组：O(n)
- 递归栈深度：O(n)

## 位运算优化

对于较大的n，可以使用位运算优化对角线判断：

```c
#include <stdio.h>

int n;
int count = 0;

void dfs(int col, int diag1, int diag2) {
    if (col == (1 << n) - 1) {
        count++;
        return;
    }
    int available = ((1 << n) - 1) & ~(col | diag1 | diag2);
    while (available) {
        int pos = available & -available;
        available -= pos;
        dfs(col | pos, (diag1 | pos) << 1, (diag2 | pos) >> 1);
    }
}

int main() {
    scanf("%d", &n);
    dfs(0, 0, 0);
    printf("%d\n", count);
    return 0;
}
```

**位运算优化原理：**
- `col`：标记哪些列已被占用
- `diag1`：标记主对角线冲突（row-col相同）
- `diag2`：标记副对角线冲突（row+col相同）
- `available`：可用列的位掩码
- `pos = available & -available`：提取最低位的1

**优点：**
- 使用整数代替数组，节省空间
- 位运算速度快，适合n<=32

## 迭代实现

使用栈模拟递归过程：

```c
#include <stdio.h>
#include <stack>
using namespace std;

struct State {
    int row;
    int path[20];
    int col_used[20];
    int diag1[40];
    int diag2[40];
    int col;
};

void nqueen_iterative(int n) {
    stack<State> st;
    State initial;
    initial.row = 1;
    initial.col = 0;
    for (int i = 0; i <= n; i++) {
        initial.col_used[i] = 0;
        initial.path[i] = 0;
    }
    for (int i = 0; i <= 2 * n; i++) {
        initial.diag1[i] = 0;
        initial.diag2[i] = 0;
    }
    st.push(initial);
    
    while (!st.empty()) {
        State curr = st.top();
        st.pop();
        
        if (curr.row == n + 1) {
            for (int i = 1; i <= n; i++) {
                printf("%d", curr.path[i]);
            }
            printf("\n");
            continue;
        }
        
        for (int col = n; col > curr.col; col--) {
            if (!curr.col_used[col] && !curr.diag1[curr.row - col + n] && !curr.diag2[curr.row + col]) {
                State next = curr;
                next.col_used[col] = 1;
                next.diag1[curr.row - col + n] = 1;
                next.diag2[curr.row + col] = 1;
                next.path[curr.row] = col;
                next.row = curr.row + 1;
                next.col = 0;
                st.push(next);
            }
        }
    }
}

int main() {
    int n;
    scanf("%d", &n);
    nqueen_iterative(n);
    return 0;
}
```

**迭代实现原理：**
- 使用栈保存搜索状态
- 每个状态包含当前行、已选列、冲突标记和当前尝试的列

## C语言基础知识补充

### 对角线冲突判断

对角线冲突的判断方法：

```c
// 主对角线：row-col相同
// 副对角线：row+col相同

// 方法1：直接计算
for (int i = 1; i < row; i++) {
    if (path[i] == col || abs(i - row) == abs(path[i] - col)) {
        // 冲突
    }
}

// 方法2：使用数组标记
int diag1[40];  // row-col+n，范围[0, 2n]
int diag2[40];  // row+col，范围[2, 2n]
```

### 数组大小计算

对角线数组的大小需要足够大：

- `diag1[row - col + n]`：row-col的范围是[-(n-1), n-1]，加上n后范围是[1, 2n-1]
- `diag2[row + col]`：row+col的范围是[2, 2n]

所以数组大小至少需要2n+1。

### 全局变量 vs 局部变量

在递归函数中，使用全局变量可以避免参数传递：

```c
// 全局变量（代码简洁，但有副作用）
int col_used[20];
int diag1[40];
int diag2[40];

// 局部变量（更安全，但需要参数传递）
void dfs(int row, int col_used[], int diag1[], int diag2[]) {
    // ...
}
```

### 位运算基础

位运算在N皇后问题中的应用：

```c
// 设置第i位为1
col |= (1 << i);

// 检查第i位是否为1
if (col & (1 << i)) {
    // 已占用
}

// 提取最低位的1
int pos = col & -col;

// 清除最低位的1
col &= (col - 1);
```

## 总结

N皇后问题是经典的回溯算法问题，核心思想是DFS+剪枝。

**解题要点：**
1. 每行选一个列放置皇后
2. 检查列冲突和对角线冲突
3. 使用三个数组记录使用状态
4. 递归终止时输出
5. 回溯时取消标记
