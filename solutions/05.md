# 题目05：假币问题

## 题目描述

林克有12枚银币。其中有11枚真币和1枚假币。假币看起来和真币没有区别，但是重量不同。但林克不知道假币比真币轻还是重。

于是他向他朋友约珥借了一架天平，用这架天平称了这些币三次。

如果用天平称两枚硬币，发现天平平衡，说明两枚都是真的。如果用一枚真币与另一枚银币比较，发现它比真币轻或重，说明它是假币。

经过精心的设计，聪明的林克根据这三次称量结果找出假币，并且能够确定假币是轻是重。

如果给你林克的称量数据，你也可以找出假币并且确定假币是轻是重吗？（林克提供的称量数据保证一定能找出假币）。

### 输入

第一行有一个数字n，表示有n组测试用例。

对于每组测试用例：

输入有三行，每行表示一次称量的结果。林克事先将银币标号为A-L。

每次称量的结果用三个以空格隔开的字符串表示：

天平左边放置的硬币 天平右边放置的硬币 平衡状态。

其中平衡状态用`up`、`down`、或`even`表示，分别为右端高、右端低和平衡。天平左右的硬币数总是相等的。

### 输出

输出哪一个标号的银币是假币，并说明它比真币轻还是重(heavy or light)。

### 输入样例

```
1
ABCD EFGH even
ABCI EFJK up
ABIJ EFGH even
```

### 输出样例

```
K is the counterfeit coin and it is light.
```

## 题目分析

这是一道经典的逻辑推理题，需要根据三次天平称量结果找出假币并判断其轻重。

**关键信息提取：**
- 12枚银币，标号A-L，其中1枚假币，11枚真币
- 假币重量与真币不同，但不知道是轻还是重
- 三次称量，每次给出：左边硬币、右边硬币、平衡状态
- 平衡状态：up（右端高）、down（右端低）、even（平衡）
- 保证一定能找出假币

**考察要点：**
1. **逻辑推理能力**：如何根据称量结果推理出假币
2. **枚举算法**：如何枚举所有可能的假币和轻重组合
3. **验证逻辑**：如何验证某个假设是否符合所有称量结果
4. **字符串处理**：如何解析输入的称量数据

## 详细思路推演

### 第一步：理解问题

我们有12枚硬币（A-L），其中1枚是假币。假币可能比真币轻（light）或重（heavy）。我们需要根据三次称量结果找出哪枚是假币，并判断它是轻还是重。

### 第二步：理解称量结果的含义

每次称量有三种可能的结果：

| 状态 | 含义 | 推理 |
|------|------|------|
| `even` | 平衡 | 左右两边都是真币 |
| `up` | 右端高 | 左端重或右端轻 |
| `down` | 右端低 | 右端重或左端轻 |

### 第三步：思考枚举策略

最直接的思路是枚举所有可能的假币和轻重组合：

1. **假币可能是A-L中的任意一枚**：共12种可能
2. **假币可能是轻或重**：共2种可能
3. **总共有12×2=24种可能**

对于每种可能，我们验证它是否符合三次称量结果。

### 第四步：思考验证逻辑

假设某枚硬币X是假币，且是轻的（light）：

- 如果X在左边：左边会变轻，所以右端会升高（状态应为up）
- 如果X在右边：右边会变轻，所以右端会升高（状态应为up）
- 如果X不在天平上：天平应该平衡（状态应为even）

假设某枚硬币X是假币，且是重的（heavy）：

- 如果X在左边：左边会变重，所以右端会降低（状态应为down）
- 如果X在右边：右边会变重，所以右端会降低（状态应为down）
- 如果X不在天平上：天平应该平衡（状态应为even）

### 第五步：思考数据结构

我们可以将每次称量的数据存储为：
- left：左边的硬币字符串（如"ABCD"）
- right：右边的硬币字符串（如"EFGH"）
- result：称量结果（"up"、"down"、"even"）

### 第六步：思考多组测试用例处理

处理流程：
1. 读取测试用例数n
2. 循环n次：
   - 读取三次称量数据
   - 枚举所有24种可能（12枚硬币×2种轻重）
   - 验证每种可能是否符合三次称量结果
   - 输出唯一符合条件的结果

## 分步算法讲解

### 算法步骤

1. **读取测试用例数**：读取n
2. **循环处理每组测试用例**：
   - **读取三次称量数据**：存储为三个结构（left, right, result）
   - **枚举假币**：遍历A-L（字符从'A'到'L'）
   - **枚举轻重**：假设是light或heavy
   - **验证假设**：检查该假设是否符合三次称量结果
   - **找到解**：输出假币编号和轻重
3. **结束**：所有测试用例处理完毕

### 流程图

```
开始
  ↓
读取 n
  ↓
循环 n 次
  ↓
读取三次称量数据
  ↓
枚举假币 c = 'A' 到 'L'
  ↓
枚举轻重 type = light, heavy
  ↓
验证假设 (c, type) 是否符合三次称量
  ↓
假设符合? → 是 → 输出结果
  ↓ 否
继续枚举
  ↓
结束循环
  ↓
结束
```

## C语言代码

```c
#include <stdio.h>
#include <string.h>

// 验证函数：检查假设 coin 是假币，type是轻重类型（0=light, 1=heavy）是否符合称量结果
// 返回1表示符合，0表示不符合
int check(char coin, int type, char left[3][7], char right[3][7], char result[3][6]) {
    for (int i = 0; i < 3; i++) {
        int left_has = strchr(left[i], coin) != NULL;
        int right_has = strchr(right[i], coin) != NULL;
        
        if (strcmp(result[i], "even") == 0) {
            if (left_has || right_has) return 0;
        } else if (strcmp(result[i], "up") == 0) {
            if (type == 0) {
                if (!left_has && !right_has) return 0;
            } else {
                if (left_has || right_has) return 0;
            }
        } else {
            if (type == 0) {
                if (left_has || right_has) return 0;
            } else {
                if (!left_has && !right_has) return 0;
            }
        }
    }
    return 1;
}

int main() {
    int n;
    scanf("%d", &n);
    
    while (n--) {
        char left[3][7], right[3][7], result[3][6];
        
        // 读取三次称量数据
        for (int i = 0; i < 3; i++) {
            scanf("%s %s %s", left[i], right[i], result[i]);
        }
        
        // 枚举所有可能的假币和轻重
        for (char c = 'A'; c <= 'L'; c++) {
            // 假设是轻的
            if (check(c, 0, left, right, result)) {
                printf("%c is the counterfeit coin and it is light.\n", c);
                break;
            }
            // 假设是重的
            if (check(c, 1, left, right, result)) {
                printf("%c is the counterfeit coin and it is heavy.\n", c);
                break;
            }
        }
    }
    
    return 0;
}
```

## 优化算法（简化验证逻辑）

### 优化思路

可以使用一个更简洁的验证逻辑：给每枚硬币一个状态值（轻为-1，重为1，真币为0），然后根据称量结果计算差值。

### 优化后的C语言代码

```c
#include <stdio.h>
#include <string.h>

int main() {
    int n;
    scanf("%d", &n);
    
    while (n--) {
        char left[3][7], right[3][7], result[3][6];
        
        for (int i = 0; i < 3; i++) {
            scanf("%s %s %s", left[i], right[i], result[i]);
        }
        
        // 枚举每枚硬币作为假币
        for (char c = 'A'; c <= 'L'; c++) {
            // 尝试轻的情况
            int valid_light = 1;
            int valid_heavy = 1;
            
            for (int i = 0; i < 3; i++) {
                int left_count = 0, right_count = 0;
                for (int j = 0; left[i][j]; j++) {
                    if (left[i][j] == c) left_count++;
                }
                for (int j = 0; right[i][j]; j++) {
                    if (right[i][j] == c) right_count++;
                }
                
                if (strcmp(result[i], "even") == 0) {
                    if (left_count + right_count > 0) {
                        valid_light = 0;
                        valid_heavy = 0;
                    }
                } else if (strcmp(result[i], "up") == 0) {
                    if (left_count > 0 || right_count > 0) {
                        valid_heavy = 0;  // 重的假币不会导致up
                    } else {
                        valid_light = 0;  // 轻的假币不在天平上
                    }
                } else {  // down
                    if (left_count > 0 || right_count > 0) {
                        valid_light = 0;  // 轻的假币不会导致down
                    } else {
                        valid_heavy = 0;  // 重的假币不在天平上
                    }
                }
            }
            
            if (valid_light) {
                printf("%c is the counterfeit coin and it is light.\n", c);
                break;
            } else if (valid_heavy) {
                printf("%c is the counterfeit coin and it is heavy.\n", c);
                break;
            }
        }
    }
    
    return 0;
}
```

### 优化效果

- 将验证逻辑合并到主循环中，减少函数调用开销
- 使用计数方式检查假币是否在天平上，逻辑更清晰

## 常见错误与易错点

1. **理解错up/down含义**：`up`表示右端高，`down`表示右端低，不要搞反。

2. **验证逻辑错误**：轻假币在左边或右边都会导致`up`；重假币在左边或右边都会导致`down`。

3. **C语言字符串比较**：必须使用`strcmp()`，不能用`==`比较字符串内容。

4. **枚举范围错误**：硬币是A-L，共12枚，循环条件应为`c <= 'L'`。

5. **多组输入处理**：先读取测试用例数n，再循环n次读取每组数据。

## 复杂度分析

### 时间复杂度

**O(n × 24 × 3 × L)**，其中：
- n是测试用例数
- 24是12枚硬币×2种轻重
- 3是三次称量
- L是每次称量的硬币数（最多6）

实际上这是一个常数时间复杂度，因为所有参数都是固定的：
- n最多是有限的
- 12枚硬币是固定的
- 3次称量是固定的

### 空间复杂度

**O(1)**，常数空间复杂度。

- 只使用了几个字符串数组存储称量数据
- 变量数量固定，与输入规模无关

## 扩展思考

### 扩展1：更多硬币（信息论分析）

信息论告诉我们，每次称量有3种结果（左重、右重、平衡），所以n次称量最多能区分3ⁿ种情况。

| 称量次数 | 最多可区分情况数 | 最多可检测硬币数 |
|----------|------------------|------------------|
| 1 | 3 | 1枚（判断轻重） |
| 2 | 9 | 3枚（找出假币并判断轻重） |
| 3 | 27 | 12枚（题目情况） |
| 4 | 81 | 40枚 |
| 5 | 243 | 121枚 |

**公式**：最多可检测的硬币数 = (3ⁿ - 3) / 2

对于13枚硬币，需要4次称量：
- 3⁴ = 81种结果
- 13×2 = 26种情况
- 81 > 26，理论上可行

### 扩展2：信息论视角详解

**信息熵**：每次称量提供的信息量为 log₂(3) ≈ 1.585比特

**总信息量**：3次称量提供 3 × log₂(3) ≈ 4.755比特

**所需信息量**：12枚硬币×2种轻重 = 24种情况，需要 log₂(24) ≈ 4.585比特

因为 4.755 > 4.585，所以理论上可行。

### 扩展3：多枚假币

如果有2枚假币，情况变得复杂得多：
- 假币组合数：C(12, 2) = 66种
- 每种组合有4种轻重情况：(轻,轻), (轻,重), (重,轻), (重,重)
- 总情况数：66 × 4 = 264种
- 需要称量次数：3⁵ = 243 < 264 < 729 = 3⁶，需要6次称量

**算法复杂度**：O(C(n, k) × 2ᵏ × 3ᵐ)，其中k是假币数量，m是称量次数

### 扩展4：最优称量策略（信息论优化）

对于单枚假币，可以使用信息论设计最优称量策略：

**策略**：每次称量将硬币分成三组，使三组的情况数尽可能相等。

例如，12枚硬币第一次称量：
- 左盘：1,2,3,4
- 右盘：5,6,7,8
- 剩余：9,10,11,12

根据结果缩小范围：
- 平衡：假币在9-12中
- 左重：假币在1-4（重）或5-8（轻）
- 右重：假币在1-4（轻）或5-8（重）

### 扩展5：找出假币但不判断轻重

如果只需要找出假币，不需要判断轻重，可以检测更多硬币：

**公式**：最多可检测的硬币数 = (3ⁿ - 1) / 2

| 称量次数 | 最多可检测硬币数 |
|----------|------------------|
| 1 | 1枚 |
| 2 | 4枚 |
| 3 | 13枚 |
| 4 | 40枚 |

### 扩展6：交互式解题

可以编写一个交互式程序，让用户输入称量结果，程序逐步推断：

```c
#include <stdio.h>
#include <string.h>

char left[3][7], right[3][7], result[3][6];

void solve() {
    for (char c = 'A'; c <= 'L'; c++) {
        int valid_light = 1, valid_heavy = 1;
        
        for (int i = 0; i < 3; i++) {
            int l_has = strchr(left[i], c) != NULL;
            int r_has = strchr(right[i], c) != NULL;
            
            if (strcmp(result[i], "even") == 0) {
                if (l_has || r_has) { valid_light = 0; valid_heavy = 0; }
            } else if (strcmp(result[i], "up") == 0) {
                if (l_has || r_has) valid_heavy = 0;
                else valid_light = 0;
            } else {
                if (l_has || r_has) valid_light = 0;
                else valid_heavy = 0;
            }
        }
        
        if (valid_light) {
            printf("%c is light\n", c);
            return;
        }
        if (valid_heavy) {
            printf("%c is heavy\n", c);
            return;
        }
    }
    printf("No solution found\n");
}

int main() {
    printf("Enter 3 weighings (left right result):\n");
    for (int i = 0; i < 3; i++) {
        scanf("%s %s %s", left[i], right[i], result[i]);
    }
    solve();
    return 0;
}
```

## C语言基础知识补充

### strchr函数详解

`strchr`是C标准库中的字符串查找函数，原型为：

```c
char *strchr(const char *str, int c);
```

**功能**：在字符串`str`中查找字符`c`，返回第一次出现的位置指针，未找到返回NULL。

**使用示例**：
```c
char str[] = "ABCD";
char *pos = strchr(str, 'C');  // pos指向'C'的位置
if (strchr(str, 'X') == NULL) {
    printf("X not found\n");
}
```

### strcmp函数详解

`strcmp`是C标准库中的字符串比较函数，原型为：

```c
int strcmp(const char *str1, const char *str2);
```

**返回值**：
- 负数：str1 < str2（字典序）
- 0：str1 == str2
- 正数：str1 > str2

**注意**：不能用`==`比较字符串内容！

```c
// 错误！比较的是指针地址
if (result[i] == "even") { ... }

// 正确！比较的是字符串内容
if (strcmp(result[i], "even") == 0) { ... }
```

### 字符数组与字符串

在C语言中，字符串是以`'\0'`结尾的字符数组：

```c
char str[6];      // 最多存储5个字符 + '\0'
str[0] = 'A';
str[1] = 'B';
str[2] = '\0';    // 必须添加终止符

// 等价于
char str[] = "AB";
```

**输入输出**：
```c
char str[100];
scanf("%s", str);   // 读取字符串（自动添加'\0'）
printf("%s", str);  // 输出字符串
```
