# 题目28：大数排序

## 题目描述

小鲁虽然编程能力很一般，但是嘴皮子上不服输。他很喜欢拿自己不懂的问题考小华。

刚学会冒泡排序的他，决定挑战一下小华的智商：

给定你一个长度为n的整数数列。

请你对这个数列按照从小到大进行排序。

并将排好序的数列按顺序输出。

小鲁刻意隐瞒了数据的规模，你觉得小华能够搞定吗？

偷偷告诉你：1≤n≤1000000，所有整数均在1~10^9范围内。

后记：小华使用快速排序秒过，小鲁惨败，因为他看不懂小华的代码......

### 输入

输入共两行，第一行包含整数n。

第二行包含n个整数，表示整个数列。

### 输出

输出共一行，包含n个整数，表示排好序的数列。

```

111584322 907287398 13562672 615771042 83035480 323016588 501254568 158361642 388135 644 58523696 400984710 38908140
```

### 输出样例 1

## 题目分析

这是一道经典的排序题目，主要考察以下几个方面：

1. **排序算法选择**：对于n=100万的数据，必须使用O(n log n)的排序算法
2. **大数据处理**：处理大规模数据的输入输出效率
3. **内存管理**：100万个整数需要约4MB内存（int）或8MB内存（long long）
4. **算法实现**：正确实现快速排序或使用语言内置排序函数

**关键信息提取：**
- 输入：n个整数（1≤n≤1000000，数值范围1~10^9）
- 输出：从小到大排序后的数列
- 时间限制：需要高效的排序算法

## 详细思路推演

### 第一步：理解问题

需要对100万个整数进行排序，数值范围1~10^9。

### 第二步：思考排序算法选择

- **冒泡排序**：O(n^2)，对于n=100万，约10^12次操作，无法通过
- **选择排序**：O(n^2)，同样无法通过
- **插入排序**：O(n^2)，同样无法通过
- **快速排序**：O(n log n)，平均约100万 × 20 = 2000万次操作，可以通过
- **归并排序**：O(n log n)，需要额外的O(n)空间
- **堆排序**：O(n log n)，原地排序
- **计数排序**：O(n + k)，k=10^9，空间太大不可行

### 第三步：思考实现方式

- C语言：实现快速排序或使用qsort函数
- C++：使用sort函数（基于快速排序的优化实现）

### 第四步：思考输入输出优化

- C语言：使用scanf读取，printf输出
- C++：使用ios::sync_with_stdio(false)加速，使用cin/cout或scanf/printf

### 第五步：思考边界情况

- n=1：只有一个元素，直接输出
- 所有元素相同：直接输出
- 逆序排列：需要完整排序

## 分步算法讲解

### 算法步骤

1. **读取输入**：读取n和n个整数
2. **排序**：使用快速排序对数组排序
3. **输出**：按顺序输出排序后的数组

### 快速排序原理

快速排序采用分治策略：

```
function quicksort(arr, left, right):
    if left >= right:
        return
    
    pivot = arr[(left + right) / 2]
    i = left, j = right
    
    while i <= j:
        while arr[i] < pivot: i++
        while arr[j] > pivot: j--
        
        if i <= j:
            swap(arr[i], arr[j])
            i++, j--
    
    quicksort(arr, left, j)
    quicksort(arr, i, right)
```

### 流程图

```
开始
  ↓
读取n
  ↓
读取n个整数到数组arr
  ↓
调用 quicksort(arr, 0, n-1)
  ↓
输出排序后的数组
  ↓
结束
```

### 算法示例

以输入 [3, 1, 4, 1, 5, 9] 为例：

```
初始数组: [3, 1, 4, 1, 5, 9]
选择pivot = 4

第一次划分:
  i从左找 >= 4 的元素 → 找到4（索引2）
  j从右找 <= 4 的元素 → 找到1（索引3）
  交换 → [3, 1, 1, 4, 5, 9]
  
继续划分，最终数组变为:
  [1, 1, 3, 4, 5, 9]

递归处理左半部分 [1, 1, 3] 和右半部分 [5, 9]
```

## C语言代码

    return *(int*)a - *(int*)b;

    int n;

        }

## C语言快速排序手动实现

```c
#include <stdio.h>
#include <stdlib.h>

void quicksort(int* arr, int left, int right) {
    if (left >= right) return;
    
    int pivot = arr[(left + right) / 2];
    int i = left, j = right;
    
    while (i <= j) {
        while (arr[i] < pivot) i++;
        while (arr[j] > pivot) j--;
        
        if (i <= j) {
            int temp = arr[i];
            arr[i] = arr[j];
            arr[j] = temp;
            i++;
            j--;
        }
    }
    
    quicksort(arr, left, j);
    quicksort(arr, i, right);
}

int main() {
    int n;
    scanf("%d", &n);
    
    int* arr = (int*)malloc(n * sizeof(int));
    for (int i = 0; i < n; i++) {
        scanf("%d", &arr[i]);
    }
    
    quicksort(arr, 0, n - 1);
    
    for (int i = 0; i < n; i++) {
        printf("%d", arr[i]);
        if (i < n - 1) {
            printf(" ");
        }
    }
    printf("\n");
    
    free(arr);
    return 0;
}
```

## 常见错误与易错点

1. **使用O(n^2)算法**：冒泡排序、选择排序、插入排序对于n=100万会超时。

2. **内存不足**：使用静态数组 `int arr[1000000]` 可能导致栈溢出，应使用动态分配或vector。

3. **输入输出效率**：对于100万数据，cin/cout速度较慢，需要使用ios::sync_with_stdio(false)或scanf/printf。

4. **比较函数错误**：qsort的比较函数必须返回int，不能直接返回差值（可能溢出）。

5. **整数溢出**：数值范围1~10^9，使用int足够（int通常为32位，范围-2^31~2^31-1）。

6. **递归深度**：快速排序最坏情况下递归深度为O(n)，可能导致栈溢出，需要优化pivot选择。

7. **输出格式错误**：最后一个数字后面不能有空格。

8. **内存泄漏**：动态分配的内存需要释放。

## 复杂度分析

### 时间复杂度

**O(n log n)**

- 快速排序平均情况下每次划分将数组分为两部分
- 递归深度为O(log n)
- 每层需要O(n)的比较操作
- 总时间复杂度为O(n log n)

### 空间复杂度

**O(log n)**（递归栈）

- 快速排序是原地排序，不需要额外的数组空间
- 递归调用栈的深度为O(log n)
- 如果使用动态分配，空间复杂度为O(n)用于存储数组

## 扩展思考

### 扩展1：三路快速排序（处理大量重复元素）

当数组中有大量重复元素时，三路快速排序效率更高：

```cpp
#include <iostream>
#include <vector>
#include <cstdlib>
#include <ctime>
using namespace std;

void quicksort3way(vector<int>& arr, int left, int right) {
    if (left >= right) return;
    
    int pivot = arr[left + rand() % (right - left + 1)];
    int lt = left, gt = right;
    int i = left;
    
    while (i <= gt) {
        if (arr[i] < pivot) {
            swap(arr[lt++], arr[i++]);
        } else if (arr[i] > pivot) {
            swap(arr[i], arr[gt--]);
        } else {
            i++;
        }
    }
    
    quicksort3way(arr, left, lt - 1);
    quicksort3way(arr, gt + 1, right);
}

int main() {
    srand(time(nullptr));
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n;
    cin >> n;
    
    vector<int> arr(n);
    for (int i = 0; i < n; i++) {
        cin >> arr[i];
    }
    
    quicksort3way(arr, 0, n - 1);
    
    for (int i = 0; i < n; i++) {
        cout << arr[i] << (i < n - 1 ? " " : "\n");
    }
    
    return 0;
}
```

### 扩展2：归并排序（稳定排序）

归并排序是稳定排序，适合需要保持相对顺序的场景：

```cpp
#include <iostream>
#include <vector>
using namespace std;

void merge(vector<int>& arr, int left, int mid, int right) {
    vector<int> temp(right - left + 1);
    int i = left, j = mid + 1, k = 0;
    
    while (i <= mid && j <= right) {
        if (arr[i] <= arr[j]) {
            temp[k++] = arr[i++];
        } else {
            temp[k++] = arr[j++];
        }
    }
    
    while (i <= mid) {
        temp[k++] = arr[i++];
    }
    
    while (j <= right) {
        temp[k++] = arr[j++];
    }
    
    for (i = left, k = 0; i <= right; i++, k++) {
        arr[i] = temp[k];
    }
}

void mergesort(vector<int>& arr, int left, int right) {
    if (left >= right) return;
    
    int mid = (left + right) / 2;
    mergesort(arr, left, mid);
    mergesort(arr, mid + 1, right);
    merge(arr, left, mid, right);
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n;
    cin >> n;
    
    vector<int> arr(n);
    for (int i = 0; i < n; i++) {
        cin >> arr[i];
    }
    
    mergesort(arr, 0, n - 1);
    
    for (int i = 0; i < n; i++) {
        cout << arr[i] << (i < n - 1 ? " " : "\n");
    }
    
    return 0;
}
```

### 扩展3：桶排序（当数值范围较小时）

当数值范围较小时，可以使用桶排序：

```cpp
#include <iostream>
#include <vector>
using namespace std;

const int MAX_VAL = 1000000000;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n;
    cin >> n;
    
    vector<int> arr(n);
    for (int i = 0; i < n; i++) {
        cin >> arr[i];
    }
    
    int max_val = 0, min_val = MAX_VAL;
    for (int x : arr) {
        max_val = max(max_val, x);
        min_val = min(min_val, x);
    }
    
    int bucket_size = 10000;
    int bucket_count = (max_val - min_val) / bucket_size + 1;
    vector<vector<int>> buckets(bucket_count);
    
    for (int x : arr) {
        int bucket_idx = (x - min_val) / bucket_size;
        buckets[bucket_idx].push_back(x);
    }
    
    int idx = 0;
    for (auto& bucket : buckets) {
        sort(bucket.begin(), bucket.end());
        for (int x : bucket) {
            arr[idx++] = x;
        }
    }
    
    for (int i = 0; i < n; i++) {
        cout << arr[i] << (i < n - 1 ? " " : "\n");
    }
    
    return 0;
}
```

## 总结

这道题是经典的排序题目，核心在于选择合适的排序算法处理大规模数据。

**核心要点：**

1. **算法选择**：必须使用O(n log n)的排序算法
2. **输入输出优化**：对于100万数据，需要高效的输入输出方式
3. **内存管理**：使用动态分配或vector避免栈溢出
4. **边界处理**：注意最后一个元素后面不能有空格

**学习收获：**

1. 理解了不同排序算法的时间复杂度
2. 掌握了快速排序的实现方法
3. 学会了处理大规模数据的技巧
4. 理解了输入输出优化的重要性

对于n=100万的数据，使用语言内置的排序函数（如qsort、sort）是最可靠的选择，它们通常经过高度优化，能够高效处理大规模数据。
