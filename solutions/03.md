# 题目03：人的周期

## 题目描述

据说人生来就有三个生理周期，分别为体力周期、感情周期和智力周期，它们的周期长度分别为23天、28天和33天。

每一个周期中有一天是高峰。在高峰这天，人会在相应的方面表现出色。例如，在智力周期的高峰，人会思维敏捷，注意力容易高度集中。

因为三个周期的长度不同，所以通常三个周期的高峰不会落在同一天。

对于每个人，想知道何时三个高峰落在同一天。

对于每个周期，会给出从当前年份的第一天开始，到出现高峰的天数（不一定是第一次高峰出现的时间）。

给定一个从当年第一天开始的天数，你的任务是输出从给定时间开始（不包括给定时间），下一次三个高峰落在同一天的时间（距给定时间的天数）。例如：给定时间为10，出现三个高峰同一天的时间是12，则输出2（注意这里不是3）。

### 输入

输入包含多组数据，每一组数据由四个整数组成，数据以 `-1 -1 -1 -1` 结束。

对于每一行的四个整数 p, e, i 和 d，其中 p, e, i 分别表示体力、情感和智力高峰出现的时间（时间从当年的第一天开始计算）。

d 是给定的时间，可能小于 p, e 或 i。所有给定时间是非负的并且小于或等于365，所求的时间小于或等于21252。

### 输出

从给定时间起，下一次三个高峰同一天的时间（距离给定时间的天数）。

输出格式为：
```
Case X: the next triple peak occurs in Y days.
```

### 输入样例

```
0 0 0 0
0 0 0 100
5 20 34 325
5 20 34 7
4 5 6 7
283 102 23 320
203 301 203 40
203 301 203 40
-1 -1 -1 -1
```

### 输出样例

```
Case 1: the next triple peak occurs in 21252 days.
Case 2: the next triple peak occurs in 21152 days.
Case 3: the next triple peak occurs in 19575 days.
Case 3: the next triple peak occurs in 19574 days.
Case 4: the next triple peak occurs in 16994 days.
Case 5: the next triple peak occurs in 8910 days.
Case 6: the next triple peak occurs in 10789 days.
Case 6: the next triple peak occurs in 10789 days.
```

## 题目分析

这是一道经典的周期性问题，需要找到三个周期的共同高峰日。

**关键信息提取：**
- 三个周期：体力(23天)、情感(28天)、智力(33天)
- 输入：p, e, i 分别是三个周期高峰出现的时间；d 是给定时间
- 输出：从 d+1 开始，下一次三个高峰同时出现的天数
- 数据范围：d ≤ 365，结果 ≤ 21252（LCM(23, 28, 33)）
- 多组输入，以 -1 -1 -1 -1 结束

**考察要点：**
1. **周期性问题**：如何理解和处理周期性
2. **枚举算法**：如何高效地找到满足条件的日期
3. **多组输入处理**：如何处理多组测试数据
4. **边界情况**：如何处理 d 小于 p, e, i 的情况

## 详细思路推演

### 第一步：理解周期性

每个人有三个生理周期：
- **体力周期**：每23天一个循环，高峰日为 p, p+23, p+46, ...
- **情感周期**：每28天一个循环，高峰日为 e, e+28, e+56, ...
- **智力周期**：每33天一个循环，高峰日为 i, i+33, i+66, ...

我们需要找到一个日期 x，使得 x 同时是三个周期的高峰日。

### 第二步：理解高峰日的判断条件

对于某一天 x，如果满足以下三个条件，则 x 是三个周期的共同高峰日：
1. (x - p) % 23 == 0 → x 是体力高峰日
2. (x - e) % 28 == 0 → x 是情感高峰日
3. (x - i) % 33 == 0 → x 是智力高峰日

这里需要注意：
- 如果 p = 0，表示高峰出现在第0天（当年第一天）
- x 必须大于 d（不包括给定时间）

### 第三步：思考枚举策略

最直接的思路是从 d+1 开始，依次检查每一天是否满足条件，直到找到第一个满足条件的 x。

枚举范围：
- 起始点：d + 1
- 终止点：d + 21252（注意是相对值，不是绝对值21252）

为什么是 `d + 21252`？因为：
- LCM(23, 28) = 23 × 28 = 644（23和28互质）
- LCM(644, 33) = 644 × 33 = 21252（644和33互质）

所以每21252天，三个周期会同时回到高峰。这个周期是**相对于起始点的**，所以枚举范围应该从 d+1 到 d+21252。

### 第四步：思考优化策略

直接枚举每一天的时间复杂度是 O(N)，其中 N 最多为 21252，这在计算机能力范围内。

但我们可以进行优化：
1. **先找体力和情感的共同高峰**：体力和情感的共同周期是 644 天
2. **在共同高峰中找智力高峰**：只需在这些特定的日子中检查智力周期

这样可以将枚举次数从最多 21252 次减少到最多 33 次。

### 第五步：思考多组输入处理

题目要求处理多组输入，直到遇到 -1 -1 -1 -1。

处理流程：
1. 读取 p, e, i, d
2. 如果四个数都是 -1，结束程序
3. 否则，计算结果并输出
4. 重复步骤1-3

### 第六步：思考输出格式

输出格式为：
```
Case X: the next triple peak occurs in Y days.
```

其中：
- X 是测试用例编号，从1开始递增
- Y 是从 d 到 x 的天数，即 x - d

## 分步算法讲解

### 算法步骤

1. **初始化**：设置测试用例编号 caseNo = 1
2. **读取输入**：读取 p, e, i, d（检查scanf返回值是否为4）
3. **检查结束条件**：如果 p == -1 && e == -1 && i == -1 && d == -1，跳出循环
4. **枚举日期**：从 d+1 到 d+21252 遍历 day（注意是相对值）
5. **检查条件**：如果 (day-p)%23==0 && (day-e)%28==0 && (day-i)%33==0，找到解
6. **输出结果**：输出 "Case X: the next triple peak occurs in Y days."，编号递增
7. **重复**：回到步骤2


## C语言代码

```c
#include <stdio.h>

int main() {
    int p, e, i, d;           // p:体力高峰, e:情感高峰, i:智力高峰, d:给定时间
    int caseNo = 1;           // 测试用例编号，从1开始
    
    // 循环读取输入，scanf返回成功读取的项数
    // == 4 表示成功读取了4个整数
    while (scanf("%d %d %d %d", &p, &e, &i, &d) == 4) {
        // 检查是否是结束标志
        if (p == -1 && e == -1 && i == -1 && d == -1) {
            break;
        }
        
        // 从 d+1 开始枚举，最多枚举到 d+21252
        // 注意：枚举范围是相对于d的，不是绝对值21252
        // 因为共同高峰周期21252天是相对于起始点的周期
        for (int day = d + 1; day <= d + 21252; day++) {
            // 检查 day 是否同时是三个周期的高峰日
            // (day - p) % 23 == 0 → day 是体力高峰日
            // (day - e) % 28 == 0 → day 是情感高峰日
            // (day - i) % 33 == 0 → day 是智力高峰日
            if ((day - p) % 23 == 0 && (day - e) % 28 == 0 && (day - i) % 33 == 0) {
                // 找到解，输出结果并编号递增
                printf("Case %d: the next triple peak occurs in %d days.\n", caseNo++, day - d);
                break;
            }
        }
    }
    
    return 0;                 // 程序正常结束
}
```

## 优化算法（减少枚举次数）

### 优化思路

可以先找到体力和情感的共同高峰，再在这些日子中找智力高峰，这样可以减少枚举次数。

**步骤：**
1. 找到体力和情感的共同高峰日（周期为 644 天）
2. 在这些共同高峰日中，找到同时也是智力高峰的日子

### 优化后的C语言代码

```c
#include <stdio.h>

int main() {
    int p, e, i, d;
    int caseNo = 1;
    
    while (scanf("%d %d %d %d", &p, &e, &i, &d) == 4) {
        if (p == -1 && e == -1 && i == -1 && d == -1) {
            break;
        }
        
        // 先找到体力和情感的共同高峰
        int day;
        for (day = d + 1; day <= d + 21252; day++) {
            if ((day - p) % 23 == 0 && (day - e) % 28 == 0) {
                break;
            }
        }
        
        // 从这个共同高峰开始，每隔 644 天检查一次智力高峰
        // 644 = LCM(23, 28)
        while (day <= d + 21252) {
            if ((day - i) % 33 == 0) {
                printf("Case %d: the next triple peak occurs in %d days.\n", caseNo++, day - d);
                break;
            }
            day += 644;  // 体力和情感的共同周期
        }
    }
    
    return 0;
}
```

### 优化效果

- 原始算法：最多枚举 21252 次
- 优化算法：最多枚举 33 次（因为每隔 644 天检查一次，21252 / 644 = 33）

## 常见错误与易错点

1. **高峰日判断条件错误**：p, e, i 是高峰出现的天数，不是余数。应该用 `(x-p) % 23 == 0` 判断，而不是 `x % 23 == p`。

2. **包含了给定时间 d**：题目要求从给定时间开始（不包括给定时间），所以应该从 `d+1` 开始枚举，不是从 `d` 开始。

3. **枚举范围错误**：21252是周期长度（相对值），不是绝对天数。应该枚举到 `d + 21252`，而不是 `21252`，否则当 d 较大时会漏掉解。

4. **多组输入处理错误**：应该用 `scanf` 的返回值判断是否结束，遇到四个 -1 时退出循环。

5. **输出格式错误**：注意题目要求的输出格式，包括大小写、空格、句点等，必须严格匹配。

## 复杂度分析

### 时间复杂度

- 最坏情况下需要枚举到 21252 天

### 空间复杂度

## 扩展思考

### 扩展1：使用中国剩余定理

这道题可以用中国剩余定理来求解，将时间复杂度降低到 O(1)。

中国剩余定理可以解决形如以下的同余方程组：
```
x ≡ p (mod 23)
x ≡ e (mod 28)
x ≡ i (mod 33)
```

由于 23、28、33 两两互质，可以直接应用中国剩余定理。

```cpp
#include <iostream>
using namespace std;

int main() {
    int p, e, i, d;
    int caseNo = 1;

    while (cin >> p >> e >> i >> d) {
        if (p == -1 && e == -1 && i == -1 && d == -1) {
            break;
        }

        // 使用中国剩余定理求解
        // M = 23 * 28 * 33 = 21252
        // M1 = 28 * 33 = 924, M2 = 23 * 33 = 759, M3 = 23 * 28 = 644
        // 求逆元：924 * 6 ≡ 1 (mod 23), 759 * 19 ≡ 1 (mod 28), 644 * 2 ≡ 1 (mod 33)

        int x = (p * 924 * 6 + e * 759 * 19 + i * 644 * 2) % 21252;

        // 处理x为0或负数的情况，确保结果在[1, 21252]范围内
        if (x == 0) x = 21252;

        // 如果 x <= d，加上一个周期，确保 x > d
        while (x <= d) {
            x += 21252;
        }

        cout << "Case " << caseNo++ << ": the next triple peak occurs in " << x - d << " days." << endl;
    }

    return 0;
}
```
