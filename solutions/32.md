# 题目32：求排列的逆序数

## 题目分析

| 字段 | 内容 |
|------|------|
| **题目来源** | 经典算法题 |
| **核心考点** | 逆序数计算、归并排序、树状数组、离散化 |
| **数据范围** | n ≤ 100000 |
| **时间限制** | 通常要求 O(n log n) |
| **空间限制** | 需要额外空间 |

**题目描述**：给定一个由1,2,...,n组成的排列，求它的逆序数。逆序数是指满足 j < k 且 i_j > i_k 的有序对 (i_j, i_k) 的个数。

---

## 详细思路推演

### 思路1：归并排序求逆序数

**步骤1：理解逆序数与归并排序的关系**

在归并排序的合并过程中，当从右半部分取元素时，说明左半部分剩余的所有元素都与当前元素形成逆序对。这是计算逆序数的关键。

**步骤2：合并过程中的逆序数统计**

```
合并 [2,6] 和 [1,3,4,5]：
i=0 (2), j=0 (1): 取1，此时左半部分有2个元素(2,6)，贡献2个逆序对(2,1),(6,1)
i=0 (2), j=1 (3): 取2，无贡献
i=1 (6), j=1 (3): 取3，左半部分有1个元素(6)，贡献1个逆序对(6,3)
i=1 (6), j=2 (4): 取4，贡献1个逆序对(6,4)
i=1 (6), j=3 (5): 取5，贡献1个逆序对(6,5)
i=1 (6), j=4: 取6，无贡献
总逆序对：2+1+1+1=5
```

**步骤3：递归计算总逆序数**

- 左半部分的逆序数：递归计算
- 右半部分的逆序数：递归计算
- 跨左右的逆序数：合并时统计
- 总逆序数 = 左 + 右 + 跨

**步骤4：边界处理**

- 当子数组只有一个元素时，逆序数为0
- 使用 `long long` 类型存储逆序数，避免溢出（n=1e5时最大逆序数约为5e9）

### 思路2：树状数组求逆序数

**步骤1：离散化**

如果数组元素范围很大（如1~1e9），需要先离散化到1~n的范围。

**步骤2：从右向左遍历**

```
对于排列 [2,6,3,4,5,1]：
从右向左：1,5,4,3,6,2
- 插入1，查询比1小的数：0，逆序对+0
- 插入5，查询比5小的数：1，逆序对+1
- 插入4，查询比4小的数：1，逆序对+1
- 插入3，查询比3小的数：1，逆序对+1
- 插入6，查询比6小的数：4，逆序对+4
- 插入2，查询比2小的数：1，逆序对+1
总逆序对：0+1+1+1+4+1=8
```

## 分步算法讲解

### 算法步骤（归并排序法）

```
merge_sort(arr, left, right):
    if left >= right:
        return 0
    mid = (left + right) / 2
    inv_left = merge_sort(arr, left, mid)       // 左半部分逆序数
    inv_right = merge_sort(arr, mid+1, right)   // 右半部分逆序数
    inv_cross = merge(arr, left, mid, right)    // 跨左右逆序数
    return inv_left + inv_right + inv_cross

merge(arr, left, mid, right):
    创建临时数组 temp
    i = left, j = mid + 1, k = 0
    inv_count = 0
    while i <= mid and j <= right:
        if arr[i] <= arr[j]:
            temp[k++] = arr[i++]
        else:
            temp[k++] = arr[j++]
            inv_count += mid - i + 1   // 左半部分剩余元素都与当前元素形成逆序
    while i <= mid:
        temp[k++] = arr[i++]
    while j <= right:
        temp[k++] = arr[j++]
    将 temp 复制回 arr[left...right]
    return inv_count
```

## C语言代码（归并排序法）

```c
#include <stdio.h>
#include <stdlib.h>

long long merge(int arr[], int left, int mid, int right) {
    int *temp = (int *)malloc((right - left + 1) * sizeof(int));
    int i = left, j = mid + 1, k = 0;
    long long inv_count = 0;
    
    while (i <= mid && j <= right) {
        if (arr[i] <= arr[j]) {
            temp[k++] = arr[i++];
        } else {
            temp[k++] = arr[j++];
            inv_count += (long long)(mid - i + 1);
        }
    }
    
    while (i <= mid) {
        temp[k++] = arr[i++];
    }
    
    while (j <= right) {
        temp[k++] = arr[j++];
    }
    
    for (int m = 0; m < k; m++) {
        arr[left + m] = temp[m];
    }
    
    free(temp);
    return inv_count;
}

long long merge_sort(int arr[], int left, int right) {
    if (left >= right) {
        return 0;
    }
    
    int mid = left + (right - left) / 2;
    long long inv_left = merge_sort(arr, left, mid);
    long long inv_right = merge_sort(arr, mid + 1, right);
    long long inv_cross = merge(arr, left, mid, right);
    
    return inv_left + inv_right + inv_cross;
}

int main() {
    int n;
    scanf("%d", &n);
    
    int *arr = (int *)malloc(n * sizeof(int));
    for (int i = 0; i < n; i++) {
        scanf("%d", &arr[i]);
    }
    
    long long result = merge_sort(arr, 0, n - 1);
    printf("%lld\n", result);
    
    free(arr);
    return 0;
}
```

**代码解释**：

| 函数 | 功能说明 |
|------|----------|
| `merge` | 合并两个有序子数组，同时统计跨左右的逆序对 |
| `merge_sort` | 递归排序，返回子数组的逆序数 |
| `main` | 读取输入，调用排序函数，输出结果 |

**关键点**：
- 使用 `long long` 类型存储逆序数，防止溢出
- 合并时统计逆序对：当取右半部分元素时，左半部分剩余的元素都与当前元素形成逆序对
- 时间复杂度：O(n log n)

## C语言代码（树状数组法）

```c
#include <stdio.h>
#include <stdlib.h>

#define MAXN 100005

int tree[MAXN];
int arr[MAXN];
int sorted[MAXN];

int lowbit(int x) {
    return x & (-x);
}

void update(int x, int n) {
    while (x <= n) {
        tree[x]++;
        x += lowbit(x);
    }
}

int query(int x) {
    int sum = 0;
    while (x > 0) {
        sum += tree[x];
        x -= lowbit(x);
    }
    return sum;
}

int cmp(const void *a, const void *b) {
    return *(int *)a - *(int *)b;
}

int main() {
    int n;
    scanf("%d", &n);
    
    for (int i = 0; i < n; i++) {
        scanf("%d", &arr[i]);
        sorted[i] = arr[i];
    }
    
    qsort(sorted, n, sizeof(int), cmp);
    
    for (int i = 0; i < n; i++) {
        int l = 0, r = n - 1;
        while (l < r) {
            int mid = (l + r) / 2;
            if (sorted[mid] >= arr[i]) {
                r = mid;
            } else {
                l = mid + 1;
            }
        }
        arr[i] = l + 1;
    }
    
    long long inv_count = 0;
    for (int i = n - 1; i >= 0; i--) {
        inv_count += query(arr[i] - 1);
        update(arr[i], n);
    }
    
    printf("%lld\n", inv_count);
    
    return 0;
}
```

**代码解释**：

| 函数 | 功能说明 |
|------|----------|
| `lowbit` | 计算最低位的1 |
| `update` | 更新树状数组 |
| `query` | 查询前缀和 |
| `main` | 离散化数组，从右向左遍历计算逆序数 |

**关键点**：
- 使用树状数组维护前缀和
- 从右向左遍历，查询比当前元素小的数的个数
- 时间复杂度：O(n log n)

## C++代码（归并排序法）

```cpp
#include <iostream>
#include <vector>

using namespace std;

long long merge(vector<int>& arr, int left, int mid, int right) {
    vector<int> temp;
    temp.reserve(right - left + 1);
    
    int i = left, j = mid + 1;
    long long inv_count = 0;
    
    while (i <= mid && j <= right) {
        if (arr[i] <= arr[j]) {
            temp.push_back(arr[i++]);
        } else {
            temp.push_back(arr[j++]);
            inv_count += (long long)(mid - i + 1);
        }
    }
    
    while (i <= mid) {
        temp.push_back(arr[i++]);
    }
    
    while (j <= right) {
        temp.push_back(arr[j++]);
    }
    
    for (int m = 0; m < temp.size(); m++) {
        arr[left + m] = temp[m];
    }
    
    return inv_count;
}

long long merge_sort(vector<int>& arr, int left, int right) {
    if (left >= right) {
        return 0;
    }
    
    int mid = left + (right - left) / 2;
    long long inv_left = merge_sort(arr, left, mid);
    long long inv_right = merge_sort(arr, mid + 1, right);
    long long inv_cross = merge(arr, left, mid, right);
    
    return inv_left + inv_right + inv_cross;
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n;
    cin >> n;
    
    vector<int> arr(n);
    for (int i = 0; i < n; i++) {
        cin >> arr[i];
    }
    
    long long result = merge_sort(arr, 0, n - 1);
    cout << result << endl;
    
    return 0;
}
```

**代码解释**：

使用 `vector<int>` 自动管理内存，无需手动释放。

## 常见错误与易错点

| 错误类型 | 错误描述 | 错误示例 | 修正方法 |
|----------|----------|----------|----------|
| **整数溢出** | 逆序数超过 `int` 范围 | `int inv_count = 0;` | 使用 `long long` |
| **离散化错误** | 未正确映射元素 | 直接使用原始值作为树状数组下标 | 先排序，再二分查找映射 |
| **合并方向错误** | 从左向右遍历时统计方式错误 | 在错误时机增加逆序对计数 | 取右半元素时统计 |
| **边界条件错误** | `mid - i` 而非 `mid - i + 1` | `inv_count += mid - i;` | 应为 `mid - i + 1` |
| **树状数组下标错误** | 从0开始而非从1开始 | `update(arr[i], n)` 当 `arr[i]=0` | 离散化时从1开始 |
| **递归终止条件错误** | `left > right` 而非 `left >= right` | 当 `left == right` 时继续递归 | 使用 `left >= right` |
| **排序稳定性问题** | 使用 `arr[i] < arr[j]` 影响结果 | 相等元素不产生逆序对 | 使用 `arr[i] <= arr[j]` |
| **大数据量超时** | 使用O(n²)的暴力算法 | 双重循环统计逆序对 | 使用归并排序或树状数组 |

## 复杂度分析

### 时间复杂度

| 算法 | 复杂度 | 说明 |
|------|--------|------|
| 归并排序法 | O(n log n) | 分治策略，每层合并O(n)，共log n层 |
| 树状数组法 | O(n log n) | 每次查询和更新O(log n)，共n次 |
| 暴力算法 | O(n²) | 双重循环，仅适用于n≤1000 |

### 空间复杂度

| 算法 | 空间复杂度 | 说明 |
|------|------------|------|
| 归并排序法 | O(n) | 需要临时数组存储合并结果 |
| 树状数组法 | O(n) | 需要树状数组和排序后的数组 |

## 扩展思考

### 思考1：逆序数的应用

逆序数在以下场景有应用：
- 排序算法的比较次数
- 衡量两个排列的差异度
- 检测数组是否有序
- 数据压缩中的熵编码

### 思考2：多种算法对比

| 算法 | 优点 | 缺点 |
|------|------|------|
| 归并排序法 | 代码简洁，无需离散化 | 需要额外O(n)空间 |
| 树状数组法 | 可在线查询，空间较优 | 需要离散化处理 |
| 线段树法 | 功能更强大 | 代码复杂，常数较大 |

### 思考3：多维逆序对

可以扩展到二维逆序对问题，即统计满足 i < j 且 a_i > a_j 且 b_i > b_j 的有序对。这可以通过CDQ分治或嵌套树状数组解决。

## 总结

求排列的逆序数是一个经典的算法问题，主要有以下两种高效解法：

**核心要点**：
- 归并排序法：在合并过程中统计逆序对，代码简洁直观
- 树状数组法：从右向左遍历，利用前缀和查询逆序对，支持在线查询
- 时间复杂度均为 O(n log n)，适用于大数据量
- 使用 `long long` 防止逆序数溢出

**学习收获**：
- 理解逆序数的定义和性质
- 掌握归并排序求逆序数的方法
- 学习树状数组的基本操作和离散化技巧
- 了解算法在实际问题中的应用

**对比两种方法**：

| 特性 | 归并排序法 | 树状数组法 |
|------|------------|------------|
| 代码复杂度 | 低 | 中 |
| 空间复杂度 | O(n) | O(n) |
| 是否需要离散化 | 否 | 是 |
| 是否支持在线查询 | 否 | 是 |
| 常数大小 | 较大 | 较小 |
