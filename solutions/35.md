# 题目35：求方程的根

## 题目分析

| 字段 | 内容 |
|------|------|
| **题目来源** | 经典二分法题 |
| **核心考点** | 二分法、函数求值、精度控制 |
| **方程** | f(x) = x³ - 5x² + 10x - 80 = 0 |
| **精度要求** | 精确到小数点后9位 |
| **输入** | 无输入 |
| **输出** | 方程的根 |

**题目描述**：使用二分法求解方程 f(x) = x³ - 5x² + 10x - 80 = 0 的根，精确到小数点后9位。

---

## 详细思路推演

### 思路：二分法求方程根

**步骤1：确定根所在的区间**

首先分析函数 f(x) = x³ - 5x² + 10x - 80 的性质：
- f(0) = 0³ - 5×0² + 10×0 - 80 = -80
- f(5) = 125 - 125 + 50 - 80 = -30
- f(6) = 216 - 180 + 60 - 80 = 16

由于 f(5) = -30 < 0，f(6) = 16 > 0，且函数在区间[5,6]上连续，根据**介值定理**，方程在区间(5,6)内至少有一个根。

**步骤2：验证函数单调性**

计算导数 f'(x) = 3x² - 10x + 10
- 判别式 Δ = 100 - 120 = -20 < 0
- 由于 Δ < 0，导数 f'(x) 在整个实数域上恒大于0
- 因此 f(x) 是单调递增函数，方程只有一个根

**步骤3：二分法迭代过程**

```
初始区间: [5, 6]
迭代1: mid = 5.5, f(5.5) = -5.625, 根在(5.5, 6)
迭代2: mid = 5.75, f(5.75) ≈ 4.921875, 根在(5.5, 5.75)
迭代3: mid = 5.625, f(5.625) ≈ -0.65625, 根在(5.625, 5.75)
迭代4: mid = 5.6875, f(5.6875) ≈ 2.109375, 根在(5.625, 5.6875)
...
继续迭代直到区间长度小于 1e-9
```

**步骤4：精度控制**

当区间长度 right - left < 1e-9 时，认为达到精度要求，输出 mid = (left + right) / 2。

## C语言代码

```c
#include <stdio.h>
#include <math.h>

double f(double x) {
    return x * x * x - 5 * x * x + 10 * x - 80;
}

int main() {
    double left = 5.0, right = 6.0;
    double precision = 1e-9;
    
    while (right - left > precision) {
        double mid = (left + right) / 2;
        if (f(left) * f(mid) < 0) {
            right = mid;
        } else {
            left = mid;
        }
    }
    
    printf("%.9f\n", (left + right) / 2);
    
    return 0;
}
```

**代码解释**：

| 函数 | 功能说明 |
|------|----------|
| `f(x)` | 计算函数值 f(x) = x³ - 5x² + 10x - 80 |
| `main` | 二分法求解方程根，输出精确到9位小数 |

**关键点**：
- 使用 `double` 类型保证精度
- 循环条件 `right - left > precision` 控制精度
- `f(left) * f(mid) < 0` 判断根所在区间
- 输出格式 `%.9f` 保证9位小数

## Python代码

```python
def f(x):
    return x ** 3 - 5 * x ** 2 + 10 * x - 80

def binary_search():
    left, right = 5.0, 6.0
    precision = 1e-9
    
    while right - left > precision:
        mid = (left + right) / 2
        if f(left) * f(mid) < 0:
            right = mid
        else:
            left = mid
    
    return (left + right) / 2

result = binary_search()
print("{0:.9f}".format(result))
```

**代码解释**：

使用 Python 的格式化输出确保9位小数精度。

## 常见错误与易错点

| 错误类型 | 错误描述 | 错误示例 | 修正方法 |
|----------|----------|----------|----------|
| **区间选择错误** | 初始区间没有包含根 | `left=0, right=5` | 先计算函数值确定包含根的区间 |
| **精度不够** | 精度设置过大 | `precision = 1e-6` | 使用 `precision = 1e-9` |
| **浮点精度问题** | 使用 float 类型精度不够 | `float left = 5.0` | 使用 `double` 类型 |
| **输出格式错误** | 输出小数位数不足 | `printf("%f", result)` | `printf("%.9f", result)` |
| **循环条件错误** | 使用 `>=` 而非 `>` | `while (right - left >= precision)` | `while (right - left > precision)` |
| **函数计算错误** | 表达式写错 | `x * x * x - 5 * x + 10 * x - 80` | `x * x * x - 5 * x * x + 10 * x - 80` |
| **符号判断错误** | 使用 `f(mid) < 0` 而非乘积 | `if (f(mid) < 0)` | `if (f(left) * f(mid) < 0)` |
| **无限循环** | 精度设置过小导致无法收敛 | `precision = 1e-20` | 合理设置精度 |

## 扩展思考

### 思考1：二分法的适用条件

二分法要求：
1. 函数在区间[a,b]上连续
2. f(a) × f(b) < 0（两端点函数值异号）
3. 函数在区间内单调（保证只有一个根）

### 思考2：其他求根方法

| 方法 | 优点 | 缺点 |
|------|------|------|
| 二分法 | 简单可靠，保证收敛 | 收敛速度较慢 |
| 牛顿迭代法 | 收敛速度快（二次收敛） | 需要计算导数，可能不收敛 |
| 割线法 | 不需要计算导数 | 收敛速度比牛顿法慢 |
| 弦截法 | 不需要计算导数 | 需要两个初始点 |

### 思考3：牛顿迭代法实现

```cpp
double f(double x) {
    return x * x * x - 5 * x * x + 10 * x - 80;
}

double f_prime(double x) {
    return 3 * x * x - 10 * x + 10;
}

double newton_method(double x0) {
    double x = x0;
    for (int i = 0; i < 100; i++) {
        double fx = f(x);
        if (abs(fx) < 1e-9) break;
        x = x - fx / f_prime(x);
    }
    return x;
}
```

牛顿迭代法通常只需要5-10次迭代就能达到1e-9的精度。

## 总结

求方程的根是一个经典的数值计算问题，使用二分法可以可靠地求解。

**核心要点**：
- 二分法通过不断缩小区间来逼近方程的根
- 需要先确定包含根的初始区间
- 使用 `double` 类型保证精度
- 控制循环条件确保达到精度要求

**学习收获**：
- 理解二分法的基本原理
- 掌握函数求值和精度控制
- 了解数值计算的基本概念
- 对比不同求根方法的优劣

**对比二分法和牛顿法**：

| 特性 | 二分法 | 牛顿迭代法 |
|------|--------|------------|
| 收敛速度 | 线性收敛 | 二次收敛 |
| 需要导数 | 否 | 是 |
| 实现难度 | 简单 | 中等 |
| 收敛保证 | 保证收敛 | 可能不收敛 |
| 适用范围 | 连续函数 | 光滑函数 |

二分法虽然收敛较慢，但实现简单且保证收敛，适合作为入门学习的数值方法。
