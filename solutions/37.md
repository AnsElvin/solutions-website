# 题目37：最小预算值

## 题目分析

| 字段 | 内容 |
|------|------|
| **题目来源** | 经典二分法+贪心题 |
| **核心考点** | 二分查找、贪心策略、单调性判断 |
| **数据范围** | 1 ≤ N ≤ 100,000，1 ≤ M ≤ N，每日支出 1~10000 |
| **时间限制** | 通常要求 O(N log S)，S为总和 |
| **空间限制** | O(N) |

**题目描述**：将N天的支出分成M组（每组连续），每组预算固定为Budget，求满足条件的最小Budget值。

---

## 详细思路推演

### 思路：二分查找 + 贪心验证

**步骤1：理解问题**

我们需要找到最小的Budget，使得：
1. 每组的总支出 ≤ Budget
2. 可以将N天分成恰好M组（连续天数）

**步骤2：确定二分范围**

- **下界 left**：单日最大支出 max(X)
  - 如果Budget小于这个值，无法覆盖最大支出的那天
- **上界 right**：所有天数支出之和 sum(X)
  - 如果Budget等于这个值，所有天数可以作为一组

**步骤3：验证函数设计**

对于给定的Budget，判断最少需要分成多少组：
- 贪心策略：从左到右累加，当累加和超过Budget时，新开一组
- 返回所需组数

**步骤4：二分查找过程**

## 分步算法讲解

### 算法步骤

```
binary_search(X, N, M):
    left = max(X)
    right = sum(X)
    
    while left < right:
        mid = (left + right) / 2
        if check(X, N, mid) <= M:
            right = mid
        else:
            left = mid + 1
    
    return left

check(X, N, budget):
    groups = 1
    current_sum = 0
    
    for i from 0 to N-1:
        if current_sum + X[i] > budget:
            groups += 1
            current_sum = X[i]
        else:
            current_sum += X[i]
    
    return groups
```

### 算法流程图

```
      开始
        ↓
  left = max(X), right = sum(X)
        ↓
  left < right?
        ↓
      是
        ↓
  mid = (left + right) / 2
        ↓
  check(X, mid) <= M?
        ↓
   是 ↓   ↓ 否
right=mid  left=mid+1
        ↓
  回到循环条件
        ↓
      否
        ↓
  输出 left
        ↓
      结束
```

## C语言代码

```c
#include <stdio.h>
#include <stdlib.h>

#define MAX_N 100005

int X[MAX_N];

int check(int *X, int N, long long budget) {
    int groups = 1;
    long long current_sum = 0;
    
    for (int i = 0; i < N; i++) {
        if (current_sum + X[i] > budget) {
            groups++;
            current_sum = X[i];
        } else {
            current_sum += X[i];
        }
    }
    
    return groups;
}

int main() {
    int N, M;
    scanf("%d%d", &N, &M);
    
    long long sum = 0;
    int max_val = 0;
    for (int i = 0; i < N; i++) {
        scanf("%d", &X[i]);
        sum += X[i];
        if (X[i] > max_val) {
            max_val = X[i];
        }
    }
    
    long long left = max_val;
    long long right = sum;
    
    while (left < right) {
        long long mid = left + (right - left) / 2;
        if (check(X, N, mid) <= M) {
            right = mid;
        } else {
            left = mid + 1;
        }
    }
    
    printf("%lld\n", left);
    
    return 0;
}
```

**代码解释**：

| 函数 | 功能说明 |
|------|----------|
| `check(X, N, budget)` | 判断预算为budget时需要的最少组数 |
| `main` | 读取输入，计算上下界，二分查找最小预算 |

**关键点**：
- 使用 `long long` 防止整数溢出
- 二分查找条件 `left < right`，避免死循环
- mid 计算方式 `left + (right - left) / 2` 防止溢出
- check函数使用贪心策略，从左到右累加

## Python代码

```python
def check(X, budget):
    groups = 1
    current_sum = 0
    
    for val in X:
        if current_sum + val > budget:
            groups += 1
            current_sum = val
        else:
            current_sum += val
    
    return groups

def binary_search(X, M):
    left = max(X)
    right = sum(X)
    
    while left < right:
        mid = (left + right) // 2
        if check(X, mid) <= M:
            right = mid
        else:
            left = mid + 1
    
    return left

N, M = map(int, input().split())
X = list(map(int, input().split()))

print(binary_search(X, M))
```

## 常见错误与易错点

| 错误类型 | 错误描述 | 错误示例 | 修正方法 |
|----------|----------|----------|----------|
| **整数溢出** | sum计算溢出 | `int sum = 0;` | 使用 `long long sum = 0;` |
| **二分条件错误** | 使用 `left <= right` | `while (left <= right)` | `while (left < right)` |
| **mid计算溢出** | `(left + right) / 2` 溢出 | `long long mid = (left + right) / 2;` | `long long mid = left + (right - left) / 2;` |
| **初始值错误** | left设为0 | `left = 0;` | `left = max(X);` |
| **check函数逻辑错误** | 分组数初始化为0 | `groups = 0;` | `groups = 1;` |
| **check函数逻辑错误** | 先判断再累加 | `if (current_sum > budget)` | `if (current_sum + X[i] > budget)` |
| **返回值错误** | 返回right而非left | `return right;` | `return left;` |
| **输入输出超时** | 未使用快速IO | 直接使用cin/cout | `ios::sync_with_stdio(false); cin.tie(nullptr);` |

## 扩展思考

### 思考1：二分查找的单调性证明

为什么可以使用二分查找？
- 当Budget增大时，所需组数不会增加（单调递减）
- 当Budget减小时，所需组数不会减少（单调递增）
- 存在一个最小的Budget，使得所需组数 ≤ M

### 思考2：其他类似问题

这个模型可以解决很多类似问题：
- 分割数组的最大值（LeetCode 410）
- 传送带上的包裹（LeetCode 1011）
- 二分查找答案的模式

### 思考3：优化方向

可以使用前缀和优化check函数，但实际上贪心策略已经是O(N)最优。

## 总结

最小预算值是一个经典的二分查找问题，结合贪心策略可以高效求解。

**核心要点**：
- 使用二分查找确定最小预算值
- 贪心策略验证给定预算是否可行
- 使用 `long long` 防止整数溢出
- 正确设置二分查找的上下界和终止条件

**学习收获**：
- 理解二分查找在最优化问题中的应用
- 掌握贪心策略的设计思路
- 学会处理大数据量的输入输出
- 了解二分查找的单调性证明

**关键技巧**：
- 二分查找条件 `left < right`
- mid 计算方式 `left + (right - left) / 2`
- check函数初始分组数为1
- 使用快速IO优化输入输出

这个问题体现了"二分答案"的经典思想：当直接求解答案困难时，可以通过二分查找答案范围，并验证每个候选答案是否可行。
