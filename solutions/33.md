# 题目33：查找指定数

## 题目分析

| 字段 | 内容 |
|------|------|
| **题目来源** | 经典查找算法题 |
| **核心考点** | 哈希表查找、二分查找、大数据量查询优化 |
| **数据范围** | 1 ≤ N ≤ 100000，T次查询 |
| **时间限制** | 通常要求 O(N + T) 或 O(N log N + T log N) |
| **空间限制** | 需要额外空间存储映射关系 |

**题目描述**：给定一个数组nums（不包含重复元素），需要从nums中迅速找出target的数组下标，如果找不到请输出-1。

---

## 详细思路推演

### 思路1：哈希表法（推荐）

**步骤1：理解哈希表的作用**

哈希表可以在O(1)时间内完成查询操作，适合需要多次查询的场景。将每个元素的值作为键，下标作为值存储在哈希表中。

**步骤2：构建哈希表**

```
输入数组: [0, 1, 5, 8, 10, ...]
构建哈希表:
{
    0: 0,
    1: 1,
    5: 2,
    8: 3,
    10: 4,
    ...
}
```

**步骤3：查询过程**

```
查询target=8: 在哈希表中查找键8，返回值3
查询target=42: 在哈希表中查找键42，不存在，返回-1
```

**步骤4：边界处理**

- 数组为空时，直接返回-1
- target不在数组中时，返回-1
- 数组元素可能为负数，哈希表可以正常处理

### 思路2：二分查找法

**步骤1：排序并记录原始下标**

由于需要返回原始下标，不能直接排序原数组。需要创建一个结构体或pair数组，同时存储值和原始下标。

**步骤2：排序**

按照值的大小对pair数组进行排序。

**步骤3：二分查找**

在排序后的数组中使用二分查找找到target，然后返回其原始下标。

## 分步算法讲解

### 算法步骤（哈希表法）

```
构建哈希表:
    for i from 0 to n-1:
        hash_table[nums[i]] = i

查询:
    for each target in queries:
        if target in hash_table:
            output hash_table[target]
        else:
            output -1
```

### 算法流程图

```
输入数组 [0, 1, 5, 8, 10, ...]
         ↓
   构建哈希表
         ↓
   {0:0, 1:1, 5:2, 8:3, 10:4, ...}
         ↓
   查询 target=8
         ↓
   hash_table[8] = 3
         ↓
   输出 3
         ↓
   查询 target=42
         ↓
   42 不在哈希表中
         ↓
   输出 -1
```

## C语言代码（哈希表法）

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>

#define MAXN 100005
#define MOD 1000003

typedef struct Node {
    int key;
    int value;
    struct Node *next;
} Node;

Node *hash_table[MOD];

int hash(int key) {
    int h = key % MOD;
    if (h < 0) h += MOD;
    return h;
}

void insert(int key, int value) {
    int h = hash(key);
    Node *node = (Node *)malloc(sizeof(Node));
    node->key = key;
    node->value = value;
    node->next = hash_table[h];
    hash_table[h] = node;
}

int find(int key) {
    int h = hash(key);
    Node *p = hash_table[h];
    while (p != NULL) {
        if (p->key == key) {
            return p->value;
        }
        p = p->next;
    }
    return -1;
}

void free_table() {
    for (int i = 0; i < MOD; i++) {
        Node *p = hash_table[i];
        while (p != NULL) {
            Node *tmp = p;
            p = p->next;
            free(tmp);
        }
    }
}

int main() {
    int n;
    scanf("%d", &n);
    
    int *nums = (int *)malloc(n * sizeof(int));
    for (int i = 0; i < n; i++) {
        scanf("%d", &nums[i]);
        insert(nums[i], i);
    }
    
    int t;
    scanf("%d", &t);
    
    while (t--) {
        int target;
        scanf("%d", &target);
        printf("%d\n", find(target));
    }
    
    free(nums);
    free_table();
    return 0;
}
```

**代码解释**：

| 函数 | 功能说明 |
|------|----------|
| `hash` | 计算哈希值，处理负数 |
| `insert` | 向哈希表插入键值对 |
| `find` | 在哈希表中查找键，返回值 |
| `free_table` | 释放哈希表内存 |
| `main` | 读取输入，构建哈希表，处理查询 |

**关键点**：
- 使用链式哈希表处理冲突
- 处理负数的哈希值，确保非负
- 时间复杂度：O(N + T)

## C语言代码（二分查找法）

```c
#include <stdio.h>
#include <stdlib.h>

#define MAXN 100005

typedef struct {
    int value;
    int index;
} Pair;

int cmp(const void *a, const void *b) {
    return ((Pair *)a)->value - ((Pair *)b)->value;
}

int binary_search(Pair *arr, int n, int target) {
    int left = 0, right = n - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (arr[mid].value == target) {
            return arr[mid].index;
        } else if (arr[mid].value < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }
    return -1;
}

int main() {
    int n;
    scanf("%d", &n);
    
    Pair *arr = (Pair *)malloc(n * sizeof(Pair));
    for (int i = 0; i < n; i++) {
        scanf("%d", &arr[i].value);
        arr[i].index = i;
    }
    
    qsort(arr, n, sizeof(Pair), cmp);
    
    int t;
    scanf("%d", &t);
    
    while (t--) {
        int target;
        scanf("%d", &target);
        printf("%d\n", binary_search(arr, n, target));
    }
    
    free(arr);
    return 0;
}
```

**代码解释**：

| 函数 | 功能说明 |
|------|----------|
| `cmp` | 排序比较函数，按值排序 |
| `binary_search` | 二分查找目标值，返回原始下标 |
| `main` | 读取输入，排序，处理查询 |

**关键点**：
- 使用结构体存储值和原始下标
- 排序后使用二分查找
- 时间复杂度：O(N log N + T log N)

## C++代码（哈希表法）

```cpp
#include <iostream>
#include <unordered_map>

using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n;
    cin >> n;
    
    unordered_map<int, int> hash_map;
    for (int i = 0; i < n; i++) {
        int num;
        cin >> num;
        hash_map[num] = i;
    }
    
    int t;
    cin >> t;
    
    while (t--) {
        int target;
        cin >> target;
        if (hash_map.count(target)) {
            cout << hash_map[target] << endl;
        } else {
            cout << -1 << endl;
        }
    }
    
    return 0;
}
```

**代码解释**：

使用C++标准库的 `unordered_map`，自动处理哈希冲突和内存管理。

## C++代码（二分查找法）

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n;
    cin >> n;
    
    vector<pair<int, int>> arr(n);
    for (int i = 0; i < n; i++) {
        cin >> arr[i].first;
        arr[i].second = i;
    }
    
    sort(arr.begin(), arr.end());
    
    int t;
    cin >> t;
    
    while (t--) {
        int target;
        cin >> target;
        
        int left = 0, right = n - 1;
        int result = -1;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            if (arr[mid].first == target) {
                result = arr[mid].second;
                break;
            } else if (arr[mid].first < target) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }
        
        cout << result << endl;
    }
    
    return 0;
}
```

**代码解释**：

使用 `vector<pair<int, int>>` 存储值和原始下标，排序后二分查找。

## 常见错误与易错点

| 错误类型 | 错误描述 | 错误示例 | 修正方法 |
|----------|----------|----------|----------|
| **哈希冲突处理不当** | 未处理哈希冲突导致数据丢失 | 直接使用数组存储，不处理冲突 | 使用链式哈希表 |
| **负数哈希值** | 负数取模结果为负数 | `int h = key % MOD;` 当key为负时 | `if (h < 0) h += MOD;` |
| **数组越界** | 二分查找边界条件错误 | `while (left < right)` | `while (left <= right)` |
| **排序丢失下标** | 直接排序原数组，丢失原始下标 | `sort(nums, nums + n);` | 使用结构体或pair存储下标 |
| **重复元素处理** | 题目说不包含重复元素，但代码未处理 | 多个相同元素映射到同一个下标 | 题目保证无重复，无需处理 |
| **输入输出超时** | 使用scanf/printf处理大数据量较慢 | 未使用快速IO | 使用 `ios::sync_with_stdio(false);` |
| **内存泄漏** | C语言中哈希表节点未释放 | 分配内存后未free | 使用完后释放所有节点 |
| **哈希表大小选择** | 哈希表大小不合适导致冲突过多 | `#define MOD 100` | 使用较大的质数作为模数 |

## 扩展思考

### 思考1：哈希表与二分查找的选择

| 场景 | 推荐算法 | 原因 |
|------|----------|------|
| 查询次数较少（T较小） | 二分查找 | 预处理时间较少 |
| 查询次数较多（T较大） | 哈希表 | 查询时间更优 |
| 内存受限 | 二分查找 | 空间复杂度相同，但哈希表常数较大 |
| 需要保持数组有序 | 二分查找 | 排序后数组有序 |

### 思考2：处理重复元素

如果数组中包含重复元素，需要返回第一个出现的下标：
- 哈希表法：第一次插入时记录下标，后续相同元素不更新
- 二分查找法：找到目标后，继续向左查找第一个出现的位置

### 思考3：多种查找算法对比

| 算法 | 时间复杂度 | 空间复杂度 | 适用场景 |
|------|------------|------------|----------|
| 哈希表 | O(1) 查询 | O(n) | 需要快速查询 |
| 二分查找 | O(log n) 查询 | O(1) | 数组有序 |
| 顺序查找 | O(n) 查询 | O(1) | 数组无序且查询次数少 |
| 二叉搜索树 | O(log n) 平均 | O(n) | 需要动态插入删除 |

## 总结

查找指定数是一个经典的算法问题，主要有以下两种高效解法：

**核心要点**：
- 哈希表法：O(1)平均查询时间，适合需要多次查询的场景
- 二分查找法：O(log n)查询时间，需要先排序并记录原始下标
- 题目保证无重复元素，简化了处理逻辑

**学习收获**：
- 理解哈希表的基本原理和实现
- 掌握二分查找的实现和边界处理
- 学会根据查询次数选择合适的算法
- 了解大数据量处理的优化技巧

**对比两种方法**：

| 特性 | 哈希表法 | 二分查找法 |
|------|----------|------------|
| 预处理时间 | O(n) | O(n log n) |
| 单次查询时间 | O(1) | O(log n) |
| 总时间（T次查询） | O(n + T) | O(n log n + T log n) |
| 代码复杂度 | 低 | 中 |
| 内存使用 | 较大 | 较小 |

当查询次数较多时（如T ≥ n），哈希表法更优；当查询次数较少时，二分查找法更优。
