# 题目38：林克的蛋糕

## 题目分析

| 字段 | 内容 |
|------|------|
| **题目来源** | 经典二分法题 |
| **核心考点** | 二分查找、浮点精度控制、贪心策略 |
| **数据范围** | 1 ≤ N, F ≤ 10000，半径 1~10000 |
| **时间限制** | O(N log S)，S为最大体积 |
| **空间限制** | O(N) |

**题目描述**：有N个圆柱体蛋糕（高为1，半径不同），要分给F+1个人。每个人分到的蛋糕体积相同，且只能是同一种口味。求每个人能得到的最大蛋糕体积。

---

## 详细思路推演

### 思路：二分查找 + 贪心验证

**步骤1：理解问题**

圆柱体体积公式：V = πr²h = πr²（h=1）

我们需要找到最大的体积v，使得：
- 将所有蛋糕按体积v切割，总共能得到至少F+1份
- 每份必须来自同一个蛋糕（不能混合口味）

**步骤2：确定二分范围**

- **下界 left**：0
- **上界 right**：最大蛋糕的体积 = π × max(r)²

**步骤3：验证函数设计**

对于给定的体积v，计算总共能分成多少份：
```
count = 0
for each cake in cakes:
    count += floor(cake_volume / v)
return count >= F+1
```

**步骤4：二分查找过程**

```
输入: N=3, F=3
半径: [4, 3, 3]
蛋糕体积: [16π, 9π, 9π]
max_volume = 16π ≈ 50.265

初始区间: [0, 50.265]
mid = 25.1325
check(25.1325): 
  蛋糕1: 16π / 8π = 2份
  蛋糕2: 9π / 8π = 1份  
  蛋糕3: 9π / 8π = 1份
  总计: 4份 >= 4 → 可行，尝试更大的v
left = 25.1325

继续迭代直到精度达到1e-6
```

**步骤5：精度控制**

由于需要输出3位小数，精度设置为1e-6即可。

## 分步算法讲解

### 算法步骤

```
binary_search(cakes, N, F):
    left = 0
    right = π * max(r) * max(r)
    
    for i from 0 to 100:
        mid = (left + right) / 2
        if check(cakes, N, mid) >= F + 1:
            left = mid
        else:
            right = mid
    
    return left

check(cakes, N, v):
    count = 0
    for i from 0 to N-1:
        count += cakes[i] / v
    return count
```

### 算法流程图

```
      开始
        ↓
  left = 0, right = π * max(r)²
        ↓
  循环100次:
        ↓
  mid = (left + right) / 2
        ↓
  check(mid) >= F+1?
        ↓
   是 ↓   ↓ 否
 left=mid  right=mid
        ↓
  循环结束
        ↓
  输出 left 保留3位小数
        ↓
      结束
```

### 算法示例

**输入**：
```
3 3
4 3 3
```

**计算过程**：
- 蛋糕体积：[16π, 9π, 9π] ≈ [50.265, 28.274, 28.274]
- 总人数：F+1 = 4

**check(8π ≈ 25.133)**：
- 蛋糕1：50.265 / 25.133 ≈ 2份
- 蛋糕2：28.274 / 25.133 ≈ 1份
- 蛋糕3：28.274 / 25.133 ≈ 1份
- 总计：4份 >= 4 → 可行

**输出**：
```
25.133
```

## C语言代码

```c
#include <stdio.h>
#include <math.h>

#define MAX_N 10005
#define PI 3.141592653589793

double cakes[MAX_N];

int check(double *cakes, int N, double v) {
    int count = 0;
    for (int i = 0; i < N; i++) {
        count += (int)(cakes[i] / v);
    }
    return count;
}

int main() {
    int N, F;
    scanf("%d%d", &N, &F);
    
    double max_volume = 0;
    for (int i = 0; i < N; i++) {
        int r;
        scanf("%d", &r);
        cakes[i] = PI * r * r;
        if (cakes[i] > max_volume) {
            max_volume = cakes[i];
        }
    }
    
    double left = 0, right = max_volume;
    
    for (int i = 0; i < 100; i++) {
        double mid = (left + right) / 2;
        if (check(cakes, N, mid) >= F + 1) {
            left = mid;
        } else {
            right = mid;
        }
    }
    
    printf("%.3f\n", left);
    
    return 0;
}
```

**代码解释**：

| 函数 | 功能说明 |
|------|----------|
| `check(cakes, N, v)` | 判断体积为v时能分成多少份 |
| `main` | 读取输入，计算体积，二分查找最大体积 |

**关键点**：
- 使用 `double` 类型保证精度
- π 值定义为 `3.141592653589793`
- 循环100次保证精度（2^100 ≈ 1e30）
- 输出格式 `%.3f` 保证3位小数

## C++代码

```cpp
#include <iostream>
#include <iomanip>
#include <vector>
#include <cmath>

using namespace std;

const double PI = 3.141592653589793;

int check(const vector<double>& cakes, double v) {
    int count = 0;
    for (double cake : cakes) {
        count += (int)(cake / v);
    }
    return count;
}

int main() {
    int N, F;
    cin >> N >> F;
    
    vector<double> cakes(N);
    double max_volume = 0;
    for (int i = 0; i < N; i++) {
        int r;
        cin >> r;
        cakes[i] = PI * r * r;
        max_volume = max(max_volume, cakes[i]);
    }
    
    double left = 0, right = max_volume;
    
    for (int i = 0; i < 100; i++) {
        double mid = (left + right) / 2;
        if (check(cakes, mid) >= F + 1) {
            left = mid;
        } else {
            right = mid;
        }
    }
    
    cout << fixed << setprecision(3) << left << endl;
    
    return 0;
}
```

**代码解释**：

使用 `vector<double>` 存储蛋糕体积，`fixed` 和 `setprecision(3)` 控制输出精度。

## 常见错误与易错点

| 错误类型 | 错误描述 | 错误示例 | 修正方法 |
|----------|----------|----------|----------|
| **π值精度不够** | 使用3.14导致精度不足 | `#define PI 3.14` | `#define PI 3.141592653589793` |
| **循环次数不足** | 循环次数太少导致精度不够 | `for (int i = 0; i < 50; i++)` | `for (int i = 0; i < 100; i++)` |
| **整数溢出** | count使用int溢出 | `int count = 0;` | 使用 `long long count = 0;` |
| **精度设置错误** | 使用`>=`而非`>` | `while (right - left >= 1e-6)` | 使用固定循环次数更可靠 |
| **输出格式错误** | 输出小数位数不足 | `printf("%f", result)` | `printf("%.3f", result)` |
| **人数计算错误** | 使用F而非F+1 | `if (count >= F)` | `if (count >= F + 1)` |
| **体积计算错误** | 使用r²而非πr² | `cakes[i] = r * r;` | `cakes[i] = PI * r * r;` |
| **check函数逻辑错误** | 使用ceil而非floor | `count += ceil(cake / v);` | `count += (int)(cake / v);` |

## 扩展思考

### 思考1：为什么使用固定循环次数

浮点二分查找使用固定循环次数（如100次）比使用精度条件（如 `right - left > 1e-6`）更可靠：
- 避免因浮点精度问题导致死循环
- 100次迭代可以达到1e-30的精度

### 思考2：二分查找的单调性证明

为什么可以使用二分查找？
- 当v增大时，能分成的份数不会增加（单调递减）
- 当v减小时，能分成的份数不会减少（单调递增）
- 存在一个最大的v，使得能分成的份数 ≥ F+1

### 思考3：类似问题

这个模型可以解决很多类似问题：
- 最大化最小值问题
- 资源分配问题
- 二分查找答案的模式

## 总结

林克的蛋糕是一个经典的二分查找问题，结合贪心策略可以高效求解。

**核心要点**：
- 使用二分查找确定最大蛋糕体积
- 贪心策略验证给定体积是否可行
- 使用高精度π值保证计算准确性
- 使用固定循环次数保证精度

**学习收获**：
- 理解二分查找在最优化问题中的应用
- 掌握浮点精度控制技巧
- 学会处理大数据量的输入输出
- 了解二分查找的单调性证明

**关键技巧**：
- 使用固定循环次数（100次）保证精度
- π值使用高精度常量
- check函数使用floor计算份数
- 输出格式`%.3f`保证3位小数

这个问题体现了"二分答案"的经典思想：当直接求解答案困难时，可以通过二分查找答案范围，并验证每个候选答案是否可行。
