# 题目30：输出前k大的数

## 题目描述

给定一个数组，统计前k大的数并且把这k个数从大到小输出。

### 输入

第一行包含一个整数n，表示数组的大小。n < 100000。

第二行包含n个整数，表示数组的元素，整数之间以一个空格分开。每个整数的绝对值不超过100000000。

第三行包含一个整数k。k < n。

### 输出

从大到小输出前k大的数，每个数一行。

### 输入样例 1

```
10
4 5 6 9 8 7 1 2 3 0
5
```

### 输出样例 1

```
9
8
7
6
5
```

## 题目分析

这是一道经典的查找第k大元素并输出的题目，主要考察以下几个方面：

1. **快速选择算法**：找到第k大的数，平均O(n)时间复杂度
2. **排序算法**：对前k大的数进行排序
3. **大数据处理**：处理n=10万的数据
4. **多种算法选择**：快速选择+排序、堆排序、直接排序

**关键信息提取：**
- 输入：n个整数（n < 100000）和整数k（k < n）
- 输出：前k大的数，从大到小，每个数一行
- 时间复杂度：需要高效的算法

## 详细思路推演

### 第一步：理解问题

需要找到数组中前k大的元素，并按从大到小的顺序输出。

### 第二步：思考算法选择

**方法1：直接排序**
- 对整个数组进行降序排序
- 取前k个元素输出
- 时间复杂度：O(n log n)

**方法2：快速选择+排序**
- 使用快速选择找到第k大的数
- 筛选出大于等于该数的元素
- 对这些元素进行排序
- 时间复杂度：O(n + k log k)

**方法3：最小堆**
- 维护一个大小为k的最小堆
- 遍历数组，保持堆中只有最大的k个元素
- 最后堆中就是前k大的元素
- 时间复杂度：O(n log k)

### 第三步：思考最优方案

对于n=10万，k相对较小的情况：
- 方法2的O(n + k log k)通常最快
- 方法3的O(n log k)也很高效
- 方法1的O(n log n)虽然简单，但相对较慢

### 第四步：思考输出格式

输出前k大的数，从大到小，每个数一行。

### 第五步：思考边界情况

- k=1：只有一个最大的数
- k=n-1：除了最小的数之外的所有数
- 所有元素相同：输出任意k个

## 分步算法讲解

### 算法步骤（快速选择+排序）

1. **读取输入**：读取n、数组、k
2. **快速选择**：找到第k大的数
3. **筛选元素**：收集所有大于等于第k大的数
4. **排序**：对筛选出的元素进行降序排序
5. **输出**：输出前k个元素

### 核心算法逻辑

```
function findTopK(arr, k):
    # 找到第k大的数（索引为n-k）
    threshold = quickSelect(arr, 0, n-1, n-k)
    
    # 收集大于等于threshold的元素
    top_k = []
    for num in arr:
        if num >= threshold:
            top_k.append(num)
    
    # 降序排序
    sort(top_k, reverse=True)
    
    # 输出前k个
    for i in 0 to k-1:
        print(top_k[i])
```


## C语言代码（快速选择+排序）

```c
#include <stdio.h>
#include <stdlib.h>
#include <time.h>

void swap(int* a, int* b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

int partition(int* arr, int left, int right) {
    int pivot_idx = left + rand() % (right - left + 1);
    swap(&arr[pivot_idx], &arr[right]);
    
    int pivot = arr[right];
    int i = left;
    
    for (int j = left; j < right; j++) {
        if (arr[j] <= pivot) {
            swap(&arr[i], &arr[j]);
            i++;
        }
    }
    
    swap(&arr[i], &arr[right]);
    return i;
}

int quickSelect(int* arr, int left, int right, int k) {
    if (left == right) {
        return arr[left];
    }
    
    int pivot_idx = partition(arr, left, right);
    
    if (pivot_idx == k) {
        return arr[k];
    } else if (pivot_idx < k) {
        return quickSelect(arr, pivot_idx + 1, right, k);
    } else {
        return quickSelect(arr, left, pivot_idx - 1, k);
    }
}

int cmpDesc(const void* a, const void* b) {
    return *(int*)b - *(int*)a;
}

int main() {
    srand((unsigned int)time(NULL));
    
    int n, k;
    scanf("%d", &n);
    
    int* arr = (int*)malloc(n * sizeof(int));
    for (int i = 0; i < n; i++) {
        scanf("%d", &arr[i]);
    }
    
    scanf("%d", &k);
    
    int threshold = quickSelect(arr, 0, n - 1, n - k);
    
    int* top_k = (int*)malloc(k * sizeof(int));
    int count = 0;
    for (int i = 0; i < n; i++) {
        if (arr[i] >= threshold && count < k) {
            top_k[count++] = arr[i];
        }
    }
    
    qsort(top_k, k, sizeof(int), cmpDesc);
    
    for (int i = 0; i < k; i++) {
        printf("%d\n", top_k[i]);
    }
    
    free(arr);
    free(top_k);
    return 0;
}
```

## C++代码（最小堆实现）

```cpp
#include <iostream>
#include <vector>
#include <queue>
#include <algorithm>
using namespace std;

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    
    int n, k;
    cin >> n;
    
    vector<int> arr(n);
    for (int i = 0; i < n; i++) {
        cin >> arr[i];
    }
    
    cin >> k;
    
    priority_queue<int, vector<int>, greater<int>> min_heap;
    
    for (int num : arr) {
        if (min_heap.size() < k) {
            min_heap.push(num);
        } else if (num > min_heap.top()) {
            min_heap.pop();
            min_heap.push(num);
        }
    }
    
    vector<int> top_k;
    while (!min_heap.empty()) {
        top_k.push_back(min_heap.top());
        min_heap.pop();
    }
    
    reverse(top_k.begin(), top_k.end());
    
    for (int num : top_k) {
        cout << num << endl;
    }
    
    return 0;
}
```

## 常见错误与易错点

1. **索引错误**：第k大的数对应的索引是n-k，不是k-1。

2. **筛选条件**：应该收集大于等于threshold的元素，不是大于。

3. **输出格式**：每个数一行，不是空格分隔。

4. **排序方向**：需要降序排序，不是升序。

5. **输入输出效率**：对于n=10万的数据，需要高效的输入输出方式。

6. **内存管理**：使用动态分配或vector，避免栈溢出。

7. **重复元素**：当有多个相同的第k大元素时，需要正确处理。

8. **边界条件**：k=1或k=n-1时的特殊情况。

## 复杂度分析

### 时间复杂度

**快速选择+排序：O(n + k log k)**

- 快速选择：O(n)
- 收集元素：O(n)
- 排序：O(k log k)
- 总时间复杂度：O(n + k log k)

**最小堆：O(n log k)**

- 遍历数组：O(n)
- 每次堆操作：O(log k)
- 总时间复杂度：O(n log k)

**直接排序：O(n log n)**

- 排序：O(n log n)
- 输出：O(k)
- 总时间复杂度：O(n log n)

### 空间复杂度

**快速选择+排序：O(k)**

- 需要存储前k大的元素
- 空间复杂度：O(k)

**最小堆：O(k)**

- 堆的大小为k
- 空间复杂度：O(k)

**直接排序：O(n)**

- 需要存储整个数组
- 空间复杂度：O(n)

## 总结

这道题是经典的查找前k大元素的题目，可以使用多种算法实现。

**核心要点：**

1. **快速选择+排序**：先找到第k大的数，再筛选排序，时间复杂度O(n + k log k)
2. **最小堆**：维护k个最大元素，时间复杂度O(n log k)
3. **直接排序**：最简单但较慢，时间复杂度O(n log n)
4. **输出格式**：每个数一行，从大到小排列

**学习收获：**

1. 理解了不同算法的时间复杂度差异
2. 掌握了快速选择和堆的使用方法
3. 学会了处理大规模数据的技巧
4. 理解了算法选择对性能的影响

对于不同的k值，选择不同的算法：当k较小时，最小堆效率最高；当k较大时，快速选择+排序更优；当k接近n时，直接排序可能更简单。
